> 零基础先记住这一句：用户记住一款消费级产品，通常不是因为它功能最多，而是因为它最先把某个价值讲明白了。
> English takeaway: Users remember the clearest value promise before they remember the longest feature list.

## 用户先记住的，不会是一长串功能

很多团队写价值主张时，习惯把自己会做的事情一股脑往上堆。功能越多，看起来越像完整。可从用户视角看，他第一次接触产品时，只会问一个更简单的问题，这东西到底先帮我解决什么。如果这句话讲不清，后面的功能再多，也很难被组织成一个可信判断。

所以价值主张的起点，不是把功能表浓缩一下，而是先决定你最想让用户记住哪一个结果。结果一旦定清，很多结构问题都会跟着变简单。首页讲什么，引导先承接什么，商店截图先放哪一张，支付墙围绕什么理由展开，都会更容易收紧。

## 一条稳的价值主张，要同时回答三个问题

第一，这个产品到底解决什么问题。第二，它为什么比替代方案更值得试。第三，用户什么时候会第一次明显感到“这东西有用”。如果这三个问题只能回答第一个，产品就很容易停在空泛承诺。如果只能回答前两个，却说不清第一次价值发生在哪里，后面的引导和留存也会变得发虚。

这也是为什么价值主张不能只是一句广告文案。广告文案可以帮助传播，但产品经理更需要的是一条贯穿链路的价值句。它要能进入应用商店页，也能进入首页导语、引导流程、案例解读和付费理由里。

## 品类不是标签，它决定用户拿谁和你比较

产品一旦进入用户视野，就会被放进某个比较框里。人工智能工具通常会被拿来比较速度、能力边界和结果质量。效率工具常被比较是否顺手、是否可靠、是否能积累资产。内容产品会被比较供给和推荐，健身与教育产品则常被比较计划感、陪伴感和长期效果。你可以不喜欢某个品类标签，但用户还是会在心里把你放进去。

这也是品类策略的重要性。它不是给产品找一个好听名词，而是先搞清楚你真正会和谁竞争，用户用什么标准来判断你。如果这层没想清，团队后面的很多沟通都会摇摆。今天像工具，明天像内容，后天又像社区，最后每一层都在讲不同的故事。

## 价值节奏不同，产品结构也会跟着不同

有些产品的价值出现得非常快，例如 ChatGPT、部分效率工具和创意工具。它们更适合把“立即开始”放在前面。有些产品的价值需要时间累积，例如 Duolingo、Calm、Strava 这类长期节奏型产品。它们更需要把今天这一步和未来进展接起来。还有一些内容产品，例如 Spotify，更依赖持续供给和关系顺滑度，所以它们的价值主张不能只卖一次结果，而要卖一段更顺的长期关系。

一旦接受“价值节奏不同”，就不会再期待所有产品都长成同一套结构。引导长短、免费层厚度、会员边界、提醒频率、案例写法，都会跟着节奏一起变化。

## 验证价值主张时，最重要的是看它能不能贯穿全链路

最有效的验证，不是问用户喜不喜欢一句文案，而是看这句价值主张能不能一路成立。它能不能在应用商店里让用户理解产品，能不能在首次体验里被兑现，能不能在留存里继续成立，能不能在支付墙前变成合理理由。如果中间任何一段接不上，说明它还不够真。

对零基础读者来说，价值主张章最值得记住的一点是，别把它当作文案题。它是产品判断题。你真正要找的，是一句能把用户认知、产品结构和后续经营都绑在一起的话。

## 推荐体验 App 与体验任务

### ChatGPT
- 平台：iOS / Web
- 为什么推荐它：它适合观察一款产品怎样用极少解释把能力价值直接打出来。
- 先体验哪一步：直接开始一次真实提问。
- 重点观察什么：它最先卖的到底是模型、答案还是效率。
- 体验任务：
  - [ ] 用一句话写下你觉得它最核心的价值承诺。
  - [ ] 观察这句承诺有没有在第一次结果里被兑现。

### Spotify
- 平台：iOS / Web / Android
- 为什么推荐它：它适合看内容产品怎样把“持续供给”讲成长期价值。
- 先体验哪一步：先用免费层播放内容，再看 Premium 页面。
- 重点观察什么：它到底在卖“听歌”还是“更顺的内容关系”。
- 体验任务：
  - [ ] 写下它最像功能承诺还是生活方式承诺。
  - [ ] 观察免费层和会员层是在围绕哪一句价值句分界。

### VSCO
- 平台：iOS / Web
- 为什么推荐它：它适合观察创意产品怎样把工具价值和审美身份绑在一起。
- 先体验哪一步：浏览编辑界面和会员介绍。
- 重点观察什么：它卖的不只是滤镜，而是一种更完整的创作关系。
- 体验任务：
  - [ ] 判断它最核心的比较对象是谁。
  - [ ] 写下它的价值主张更偏结果、工具还是身份感。
