## 🧠 什么是 Markdown

`Markdown` 是一种很轻的文档格式。

你可以把它理解成：

不用像 Word 那样到处点按钮，
而是直接用一些很简单的符号，
写出标题、列表、表格、引用、代码块。

最常见的几个写法：

```md
# 一级标题
## 二级标题

- 列表项
- 列表项

> 这是一段引用

`这是行内代码`
```

文件名通常长这样：

`xxx.md`

这里的 `.md` 就是文件后缀，表示“这是一个 Markdown 文件”。

> 🌐 **In English:**
> Markdown is a lightweight writing format.
> You use simple symbols to make titles,
> lists, quotes, and other structures.

## 📚 原文 7 个阶段，用人话重讲

下面是最重要的部分。你不用背原名，先理解每一步在做什么。

### 阶段1：需求确认与用户画像

原文意思：

先问清楚“这个人是谁，要学什么，要学到什么程度”。

白话版：

如果不先搞清楚学习者是谁，后面写出来的资料很容易太难或者太浅。

比如：

- 是不是零基础
- 是想找工作还是只想了解
- 想看中文还是中英双语
- 用 `Obsidian` 看，还是用 `Notion` 看

你可以把这一阶段理解成：

先定读者，再写内容。

### 阶段2：模块结构设计

原文意思：

把整套知识库拆成很多模块和文件。

白话版：

不要想到哪写到哪。
要先想清楚“目录长什么样”。

比如可以先有：

- 导航
- 时间线
- 基础概念
- 核心知识
- 案例拆解
- 词汇表
- 学习计划

你可以把这一阶段理解成：

先搭书架，再往书架上放书。

### 阶段3：排版规范确定

原文意思：

全套资料要统一格式，不然很乱。

白话版：

你要先规定：

- 每篇开头长什么样
- 英文总结放哪里
- 表格最多几列
- 哪些写法不能用

原文特别强调两个坑：

1. 不要让文件以 `# 标题` 开头
2. 不要让文件以 `---` 开头

原因是它主要考虑 `Obsidian` 的显示效果：

- `# 标题` 开头，可能造成标题重复
- `---` 开头，可能被误认成 `YAML frontmatter`

`frontmatter` 你先理解成：

“写在文档最上方的一段特殊元信息”。

而这份工作流不希望误触发这个功能。

### 阶段4：分批内容生产

原文意思：

不要一次生成全部内容，而是分批做。

白话版：

如果你一下子做太多，容易乱，也不方便检查。

所以更稳的做法是：

先做基础部分，
再做核心部分，
再做案例和词汇表，
最后做练习和学习计划。

原文提到 `Agent`，你先把它理解成：

一个能帮你批量写文档的 AI 助手。

### 阶段5：质量审查与修复

原文意思：

内容写出来以后，不是直接完工，要检查。

白话版：

要看这些东西有没有出问题：

- 开头格式对不对
- 有没有漏掉英文段落
- 有没有出现奇怪的 HTML 写法
- 段落是不是太长
- 对小白友不友好

### 阶段6：Obsidian 适配检查

原文意思：

把写好的东西放进 `Obsidian` 里，看显示是否正常。

白话版：

你可以理解成“试运行”。

检查：

- 有没有红字报错
- 标题会不会重复
- 表格能不能正常显示
- 链接能不能点

### 阶段7：交付与记忆存档

原文意思：

最后整理结果，记录这次项目做了什么。

白话版：

做完以后，要知道：

- 一共做了多少文件
- 哪些模块完成了
- 还有什么以后要继续补

你可以把这一阶段理解成：

收尾、汇报、留档。

> 🌐 **In English:**
> The 7 stages mean:
> know the learner, design the structure,
> set writing rules, create content,
> review it, check it in Obsidian,
> and record the final result.

## 🧪 你在 VS Code 里最常用的实际动作

现在把理论压缩成操作。

### 1. 打开一个文件夹

英文你会看到：

`File` → `Open Folder...`

中文理解：

把一个文件夹整体交给 `VS Code` 管理。

不是只打开一个文件，
而是打开整套资料所在的位置。

### 2. 在左边找到文件

你要看的区域是：

`Explorer`

如果没看到，通常点左边最上面的文件图标。

### 3. 打开 `.md` 文件

你在左边点一个文件，
中间就会在 `Editor` 编辑区打开。

像下面这种文件就是 Markdown：

[`knowledge_base_workflow_general.md`](<../../Vaults/Vibe_Coding_Knowledge_Vault/workflow/knowledge_base_workflow_general.md>)

### 4. 修改内容并保存

保存的英文一般是：

`Save`

快捷键在 macOS 常见是：

`Command + S`

### 5. 看 Markdown 预览

你可能会看到：

`Open Preview`

或者带放大镜、书页、预览相关的小图标。

它的作用是：

让你看到 Markdown 渲染后的效果，
而不是只看原始符号。

### 6. 打开终端

终端英文是：

`Terminal`

你可以理解成：

一个可以输入命令的文字窗口。

原文后面出现 `bash` 命令、检查命令时，
通常就是在这个区域输入。
