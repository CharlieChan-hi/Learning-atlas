这份词表专门用来帮你看懂桌面工作区里常见的英文命名。

重点不是死记翻译，而是看懂 4 件事：

- 这个词最小可以怎么拆
- 它字面上是什么意思
- 它在产品和开发协作里通常指什么
- 你以后什么时候可以拿它来命名

---

## 1. 先记住怎么看名字

常见拆词方式：

- `snake_case`
  例子：`project_progress`
- `kebab-case`
  例子：`station-navigation-hub`
- `PascalCase`
  例子：`HealthArchiveH5Prototype`
- `camelCase`
  例子：`projectContext`
- 缩写
  例子：`src`、`api`、`cli`

看名字时，优先判断 3 件事：

1. 它是容器，还是项目，还是文档，还是代码文件
2. 它是在说内容主题，还是说阶段状态
3. 它是在说给谁看，还是说拿来做什么

## 3. 产品经理特别值得会的一组词

### `insight`

- 字面意思：洞察
- 真正工作里的意思：对用户、数据、问题做完理解后的结论
- 什么时候用：当内容已经不是原始数据，而是带解释和判断时
- 例子：`Pregnancy Insights`

### `tracker`

- 字面意思：追踪器
- 真正工作里的意思：持续记录状态变化的功能或模块
- 什么时候用：习惯追踪、进度追踪、健康追踪、问题追踪
- 例子：`Pregnancy Tracker`

### `review`

- 字面意思：复核、复看
- 真正工作里的意思：质量检查、方案评审、结果审查
- 什么时候用：需要“再看一轮”的页面、流程、内容区
- 例子：`Review`, `Code Review`

### `workflow`

- 字面意思：工作如何流动
- 真正工作里的意思：从输入到输出的一套流程组织
- 什么时候用：交付流程、AI 协作流程、内容生产流程
- 例子：`WORKFLOW.md`

### `archive`

- 字面意思：归档
- 真正工作里的意思：历史保留，不再是现行主维护面
- 什么时候用：旧版本、历史资料、快照、旧输出
- 例子：`Career Learning Archive`

### `principles`

- 字面意思：原则
- 真正工作里的意思：帮助你做判断的一组长期规则
- 什么时候用：设计原则、产品原则、团队原则
- 例子：`five_product_principles.jpeg`

### `graphics`

- 字面意思：图形视觉
- 真正工作里的意思：偏设计、插图、图解、视觉资料，而不是普通截图
- 什么时候用：视觉素材包、图解学习资料、漫画/插图集合
- 例子：`graphic_books`

## 5. 看到这些名字时，你可以怎么拆

### `Workspace_Tools`

- `Workspace` + `Tools`
- 意思：工作区工具集合
- 判断：不是普通资料夹，而是一组为整个工作区服务的工具

### `Path_Harness`

- `Path` + `Harness`
- 意思：路径治理执行壳
- 判断：是一个统一收口路径扫描、规划、验证动作的工具入口

### `HealthArchiveH5Prototype`

- `Health` + `Archive` + `H5` + `Prototype`
- 意思：健康档案方向的 H5 原型
- 判断：领域、内容主题、页面形态、成熟度一次说清楚了

### `FitnessStationDashboard`

- `Fitness` + `Station` + `Dashboard`
- 意思：健身驿站相关的总览界面
- 判断：是一个总览型页面，不是通用项目名

### `Vibe_Coding_Knowledge_Vault`

- `Vibe` + `Coding` + `Knowledge` + `Vault`
- 意思：Vibe Coding 主题的系统知识库
- 判断：这是长期知识系统，不是临时资料夹

### `Product_Manager_Interview_Kit`

- `Product` + `Manager` + `Interview` + `Kit`
- 意思：产品经理面试资料包
- 判断：`kit` 表示成组材料，不是一篇孤立文档

### `station_navigation_hub.html`

- `station` + `navigation` + `hub`
- 意思：驿站导航枢纽页
- 判断：`hub` 暗示它是入口汇聚页

### `legacy_to_unified_mapping_v1.json`

- `legacy` + `to` + `unified` + `mapping` + `v1`
- 意思：旧结构到统一结构的映射表，第 1 版
- 判断：典型工程数据文件命名

## 6. 以后可以直接复用的命名模板

### 工作区 / 总空间

- `Learning Studio`
- `Portfolio Lab`
- `Design Library`
- `Case Files`

适合：大类空间、长期整理区、策展式工作区

### 项目根 / 原型根

- `HealthArchiveH5Prototype`
- `FitnessStationDashboard`
- `Focus_Training_Platform`
- `AI_PM_Knowledge_System`

适合：现行项目、原型项目、系统型项目

### 文档名

- `local_setup_guide.md`
- `project_progress.md`
- `validation_report_v1.md`
- `workflow_muscle_gain_execution_flow.md`

适合：维护型说明文档、流程文档、同步文档

### 资源区 / 资料区

- `Assets`
- `Templates`
- `Sources`
- `References`
- `Archive`

适合：容器类目录

## 7. 简单命名规则

- 工作区和一级容器，用清楚、稳定、好读的英文名
- 现行项目根，优先用紧凑清晰的项目名，不要太口语
- Markdown 和说明文档，优先用 `snake_case`
- HTML 页面文件，优先写清“页面对象 + 页面职责”
- 不要为了整齐，把所有第三方文件名都硬翻成英文
- 一个名字最好同时回答这是什么、给谁看、用来干嘛

---

## 8. 最后一个判断标准

如果你看到一个名字，能马上回答下面 3 个问题，这个名字通常就还不错：

1. 这是项目、容器、文档，还是代码文件
2. 它在当前流程里是现行入口、预览、评审，还是归档
3. 下次我自己做类似东西时，能不能直接复用这个命名结构

如果答不出来，这个名字大概率就不够好。
