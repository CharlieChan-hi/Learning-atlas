# 延伸阅读与高质量 X 观点库

这一模块只做一件事：管理阅读来源和观点线索，帮助你知道哪些资料能支撑事实，哪些只能作为启发。

它不是章节正文模板，也不是链接收藏夹。每条来源都要说明用途和证据边界。

## 怎么读

1. 先读 `01_延伸阅读与高质量X观点库.md`，了解来源分层。
2. 再做 `02_真实案例与练习.md`，练习判断一个来源能不能支撑你的结论。
3. 写其他章节时，回到这里找来源，但不要复制这里的段落。

## 来源分层

| 来源类型 | 可以支持什么 | 不能支持什么 |
|---|---|---|
| 官方工程博客 | 公开架构事实、设计动机、历史实践 | 当前未披露内部细节 |
| 官方文档 | API、产品能力、限制、接入方式 | 私有实现、模型权重 |
| 公开论文/演讲 | 方法、实验、团队观点 | 当前线上完整系统 |
| 可信访谈/Newsletter | 观点、趋势、角色变化 | 单独证明公司内部实现 |
| X/Twitter | 线索、讨论引子 | 架构事实的唯一证据 |

## 使用规则

- 能用官方工程博客，就不要用二手总结替代。
- X/Twitter 观点必须标注为“观点线索”或“待核验”。
- 不能把“行业通常如此”写成“某公司内部就是如此”。
- 公开资料没披露的模型权重、服务数量、实时特征、组织边界，一律写未披露。
- 每章延伸阅读只保留最相关 3 到 5 个来源。

## 本模块和其他模块的关系

- `00_资料来源总表.md` 是全库来源总账。
- 本模块是阅读和观点线索库。
- 公司案例章节只引用和本章直接相关的来源。
- 方法论章节优先引用 AWS、Google SRE、OWASP、API 设计指南等方法来源。
