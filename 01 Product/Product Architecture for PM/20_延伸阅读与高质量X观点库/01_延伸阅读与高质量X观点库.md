# 延伸阅读与高质量观点库

这个文件是观点和阅读线索库，不是公司案例正文。它的作用是帮助你补充判断框架，而不是把同一段通用开场复制到每个章节。

## 怎么使用这些观点

读观点文章时，不要急着把句子搬进正文。先判断它能帮你回答什么问题：

| 问题 | 适合使用的来源 |
|---|---|
| 产品经理为什么要懂技术判断 | Lenny、Product Tech Sense、公开演讲 |
| 如何评审架构质量 | AWS Well-Architected、Google SRE |
| 如何把用户体验和可靠性连起来 | Product-Focused Reliability for SRE |
| API、权限、安全怎么问问题 | Google / Microsoft API 指南、OWASP |

观点来源只能帮助解释和组织思路；具体公司内部架构仍然要回到官方工程博客、官方文档或论文。

## X/Twitter 线索怎么处理

X/Twitter 可以提供选题和观点，但不要单独支撑事实。使用时按三步处理：

1. 先看原帖说的是观点、经验还是事实。
2. 如果是事实，必须找官方博客、文档、论文或公开演讲二次确认。
3. 找不到可验证来源，就写“待核验线索”，不要写进正文结论。

## 写进章节前的检查

- 这条观点是否真的帮助理解本章案例？
- 它是否会让章节变成泛泛而谈？
- 有没有具体事实可以支撑？
- 如果只是励志或趋势判断，是否应该留在本观点库，而不是塞进案例正文？

记住：观点库是工具箱，不是正文模板。案例章节要讲产品本身的问题。
