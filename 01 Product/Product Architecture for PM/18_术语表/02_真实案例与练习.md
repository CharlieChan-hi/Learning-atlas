# 术语表练习

## 练习目标

训练你把术语放回真实产品场景，而不是只会背定义。

## 练习 A：支付重复点击

用户点击支付后网络卡住，又连续点了两次。

请解释这些术语在这个场景里的作用：

| 术语 | 你要写清楚什么 |
|---|---|
| 幂等 | 为什么三次请求不应该变成三次扣款 |
| 重试 | 客户端和服务端分别什么时候重试 |
| 状态机 | 待确认、处理中、成功、失败如何展示 |
| Webhook | 支付机构重复通知时如何处理 |
| 错误码 | 用户和客服如何知道下一步 |

合格答案应该说明：同一次订单支付对应同一个业务意图，重复点击只能返回原处理结果或当前处理状态。

## 练习 B：Instagram Explore 新内容曝光

平台希望给新创作者更多曝光，但不能明显降低内容质量。

请解释这些术语：

| 术语 | 你要写清楚什么 |
|---|---|
| 召回 | 新创作者内容如何进入候选池 |
| 排序 | 如何避免只按老作者历史表现排序 |
| 重排 | 是否给新作者设置曝光保护 |
| 日志事件 | 曝光、停留、隐藏、举报要怎么记录 |
| 护栏指标 | 怎么证明没有伤害用户体验 |

合格答案不应该说“直接提高新作者权重”。你要说明可能误伤质量、兴趣匹配和安全治理。

## 练习 C：Airbnb 高风险预订

新注册用户临近入住预订高价房源，房东历史较少。

请解释这些术语：

| 术语 | 你要写清楚什么 |
|---|---|
| 风险评分 | 账号、交易、行为、支付信号如何参与判断 |
| 认证 | 是否需要补充身份验证 |
| 人工审核 | 哪些情况不适合自动放行或拦截 |
| 审计日志 | 后续投诉或申诉如何追溯 |
| 申诉 | 误伤正常用户后如何恢复路径 |

合格答案应该把动作拆成放行、验证、审核、限制、拦截，而不是只有通过或拒绝。

## 练习 D：把术语翻译成 PM 问题

把下面术语各写成一个评审问题：

| 术语 | 示例问题 |
|---|---|
| 缓存 | 数据过期时用户会不会看到错误价格？ |
| 队列 | 用户需要等多久才能知道任务完成？ |
| SLO | 哪个关键旅程需要稳定到什么程度？ |
| 限流 | 被限流的正常用户会看到什么提示？ |
| 最终一致 | 两个页面短时间状态不一致时怎么解释？ |

## 自测问题

1. 幂等和重试分别解决什么问题？
2. 缓存为什么既提升体验，也可能制造错误体验？
3. 推荐系统为什么不能只看点击率？
4. 风控为什么必须有解释、审核和申诉路径？
5. SLO 为什么应该围绕用户旅程，而不是只围绕服务器指标？
