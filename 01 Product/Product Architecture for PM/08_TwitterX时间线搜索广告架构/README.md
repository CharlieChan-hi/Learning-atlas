# Twitter/X 时间线、搜索、广告架构

本模块用 Twitter/X 理解实时信息网络：时间线、搜索和广告都像“候选 + 排序 + 反馈”，但用户意图和优化目标完全不同。

## 本模块要解决的问题

用户浏览时间线是在隐式表达兴趣，用户搜索是在显式表达查询，广告系统还要在商业目标和体验护栏之间取舍。系统必须处理实时发布、索引、候选召回、排序、日志反馈、安全治理和商业化。

## 阅读重点

- 时间线和搜索不能用完全一样的排序目标。
- 日志管道是推荐、广告和实验评估的基础，不是后台杂事。
- 广告排序必须有频控、低质广告过滤和用户负反馈护栏。
- 历史 Twitter Engineering 文章能支持当时公开事实和系统思想，不能直接写成 X 当前内部全貌。
