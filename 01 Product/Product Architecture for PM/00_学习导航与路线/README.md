# 学习导航与路线

这部分只负责告诉你怎么读这套资料，不讲公司案例，也不复制通用架构定义。

## 你应该怎么读

先建立问题意识，再读案例，最后用工具箱复盘。

| 阶段 | 读什么 | 目标 |
|---|---|---|
| 入门 | 01–03 | 知道架构、系统能力、数据流、风险这些词到底在产品里指什么 |
| 案例 | 04–09 | 用 Instagram、Spotify、Netflix、Uber、Twitter/X、Airbnb、Stripe 建立真实直觉 |
| 工具 | 10–17 | 学会从需求推导系统能力，并能问出评审问题 |
| 复习 | 18–20 | 查术语、做作业、回到来源验证 |

## 阅读原则

- 不背名词，先找用户承诺。
- 不看模板，先问这个产品到底难在哪里。
- 不把推断写成事实。
- 每读一章，都要能画出“用户动作 → 系统能力 → 数据对象 → 风险点 → 验证指标”。

## 推荐路线

### 4 周快速路线

第 1 周：01–03，建立基本语言。  
第 2 周：04–06，看推荐、播放、首页和实验。  
第 3 周：07–09，看实时市场、信息流、平台风控和支付。  
第 4 周：10–17，用工具箱复盘一个自己熟悉的产品。

### 8 周稳扎稳打路线

每周读 2–3 个模块。案例模块必须做练习，方法模块必须拿一个真实功能套用。

### 12 周训练营路线

每周一个主题，最后用 Uber、Netflix 或 Stripe 做一次完整架构评审。

## 读完每章的检查

- 我能不能说清这个产品的核心对象？
- 我能不能说清数据怎么流动？
- 我能不能说出 3 个取舍？
- 我能不能区分公开事实、合理推断和未披露？
