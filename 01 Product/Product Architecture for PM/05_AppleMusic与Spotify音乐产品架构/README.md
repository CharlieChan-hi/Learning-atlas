# Apple Music 与 Spotify 音乐产品架构

本模块用 Apple Music 和 Spotify 理解音乐产品：推荐不是只选几首歌，而是把个性化、播放连续性、歌词理解、设备体验和实验评估组合成“听下去”的体验。

## 本模块要解决的问题

音乐产品的难点在于用户经常不想做复杂选择，只希望当前场景下音乐自然继续。系统要理解用户偏好、播放上下文、内容特征、设备限制和反馈信号，同时避免推荐疲劳、跳歌率升高或智能功能干扰基础播放。

## 阅读重点

- Apple Music 更适合观察播放体验、歌词翻译、设备可用性和系统级体验集成。
- Spotify 更适合观察 Home 个性化、cards and shelves、用户画像和实验评估。
- 音乐推荐要同时处理“我喜欢什么”和“我现在适合听什么”。
- 公开资料能支持功能和系统思路，但不能推断私有推荐模型、权重或完整服务链路。

## 读完后应该能回答

1. 音乐首页为什么不是简单歌单列表？
2. 播放连续性为什么也是系统能力？
3. AutoMix 这类智能播放功能需要哪些失败兜底？
4. 为什么个性化系统和实验评估不能混为一谈？
