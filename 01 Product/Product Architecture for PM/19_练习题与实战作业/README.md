# 练习题与实战作业

这一模块把前面所有章节变成可交付的训练题。目标不是答对名词，而是练会从产品问题推导系统能力、数据对象、风险边界和验证指标。

## 怎么使用

1. 先读 `01_练习题与实战作业.md`，了解作业格式和评分标准。
2. 再做 `02_真实案例与练习.md`，用真实产品场景训练。
3. 每道题都必须区分：公开事实、合理推断、公开资料未披露。

## 作业提交格式

每个答案建议按 7 段写：

1. 用户场景：谁在什么情况下遇到什么问题。
2. 产品承诺：系统要保证什么体验。
3. 核心对象：订单、内容、支付意图、风险事件、通知等。
4. 系统能力：推荐、搜索、API、风控、支付、异步任务等。
5. 失败路径：超时、重复、误判、延迟、不一致、误伤等。
6. 验证指标：主指标、护栏指标、分群指标、长期指标。
7. 图示说明：画一张图说明对象、能力、数据流或状态机。

不同题型建议配不同图：

| 题型 | 推荐图 |
|---|---|
| 推荐 / 排序 | 推荐漏斗图：召回 → 排序 → 重排 → 展示 → 反馈 |
| 支付 / 订单 | 状态机或时序图：请求 → 处理中 → webhook → 最终状态 |
| 权限 / 安全 | 权限矩阵：角色 × 资源 × 动作 |
| 出行 / 实时匹配 | 数据流图：请求 → 供给 → ETA/价格 → 匹配 → 反馈 |
| API / 平台 | 协作图：前端 → API → 服务 → 数据库/第三方/队列 |

## 评分标准

| 维度 | 好答案 | 差答案 |
|---|---|---|
| 场景 | 具体到用户、时间、动作 | 只写“优化体验” |
| 对象 | 能列清状态和关系 | 只写页面元素 |
| 能力 | 能说明系统如何配合 | 堆技术名词 |
| 风险 | 包含异常和误伤 | 只写成功路径 |
| 指标 | 有主指标和护栏 | 只写 DAU、转化率 |
| 边界 | 区分事实和推断 | 把猜测写成内部事实 |

## 练习原则

- 不追求一次写完大而全方案。
- 先把用户承诺和业务对象写清楚。
- 每个技术词都要落到产品体验。
- 每个指标都要说明可能误导什么。
- 公开资料没披露的地方必须明说不知道。
