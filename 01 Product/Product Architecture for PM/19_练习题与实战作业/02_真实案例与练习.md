# 综合实战练习

## 练习 A：Netflix 新用户首页

### 场景

用户刚注册，没有观看历史，只选择了几个偏好类型。你要设计首页首屏推荐实验。

### 任务

写出：

1. 新用户可用信号。
2. 首页行结构。
3. 内容候选来源。
4. 封面图策略。
5. 主指标、护栏指标、长期指标。
6. 公开事实、合理推断、未披露。

### 参考答案骨架

- 可用信号：地区、设备、注册偏好、入口来源、首次浏览。
- 首页行结构：热门稳定入口、偏好个性化入口、探索学习入口。
- 候选来源：热门内容、近期高质量内容、偏好类型内容、探索内容。
- 封面策略：不能只追点击，要看播放后退出。
- 指标：首次播放率、首日观看、快速退出率、首周留存。

## 练习 B：Stripe 支付重试

### 场景

用户支付时网络卡住，客户端重试，服务端和 webhook 都可能收到重复请求。

### 任务

写出：

1. 业务意图和幂等键关系。
2. 支付状态机。
3. 重复请求处理。
4. webhook 重复和延迟处理。
5. 用户提示和客服兜底。
6. 指标。

### 参考答案骨架

- 同一订单支付对应一个业务意图。
- 重试必须复用同一个幂等键。
- 服务端发现同一幂等键时返回原结果或当前状态。
- webhook 只能按状态规则更新，不能重复产生副作用。
- 用户看到“处理中、成功、失败可重试、需人工处理”等明确状态。

## 练习 C：Uber 雨天晚高峰机场叫车

### 场景

雨天、周五晚高峰、机场出行需求突然增加，司机供给不足。

### 任务

写出：

1. 乘客侧承诺。
2. 司机侧承诺。
3. 平台侧目标。
4. 需求预测和供给预测需要什么信号。
5. ETA、动态定价、司机激励的取舍。
6. 失败路径和指标。

### 参考答案骨架

- 乘客关心能否叫到车、多久到、价格是否可信。
- 司机关心接单是否划算、空驶是否可控。
- 平台关心供需平衡、履约成功率、取消率和激励成本。
- 不能只优化最近司机，要看全局供需和后续订单。

## 练习 D：Airbnb 高风险预订

### 场景

新用户临近入住预订高价房源，房东历史较少。

### 任务

写出：

1. 风险系统介入事件。
2. 账号、交易、行为、支付、关系信号。
3. 风险等级到处置动作。
4. 用户解释和申诉路径。
5. 误拦和漏拦指标。

### 参考答案骨架

| 风险等级 | 处置 |
|---|---|
| 低 | 放行并记录 |
| 中 | 要求额外验证或快速审核 |
| 高 | 暂停预订并人工审核 |
| 不确定 | 限制部分动作并请求更多信息 |

关键是不能只做“通过/拒绝”，还要有验证、审核、解释、申诉和运营处理。

## 练习 E：Twitter/X 低质量内容治理

### 场景

你要减少低质量推文曝光，但不能明显降低用户发现新内容的能力。

### 任务

写出：

1. 低质量内容类型。
2. 候选、排序、重排分别如何处理。
3. 日志事件。
4. 新作者和小众内容护栏。
5. 广告质量护栏。
6. 实验指标。

### 参考答案骨架

- 低质量要拆成重复搬运、标题党、垃圾营销、低相关、风险内容。
- 候选层可过滤明显违规，排序层可降权，重排层可限制连续出现。
- 需要记录曝光、停留、点击、隐藏、举报、屏蔽、关注/取关。
- 护栏要看新作者曝光、内容多样性、留存和广告负反馈。

## 最终作业：写一份 2 页产品架构评审稿

选择一个你熟悉的产品功能，按以下结构写：

1. 背景和用户场景。
2. 产品承诺。
3. 核心对象和状态。
4. 系统能力和数据流。
5. 关键取舍。
6. 异常和兜底。
7. 指标和实验。
8. 公开事实、合理推断、未披露。

如果你的答案删掉产品名后仍看不出讲的是哪个产品，说明还太模板化，需要重写。
