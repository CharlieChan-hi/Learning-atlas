# Airbnb 与 Stripe 平台能力拆解

本模块用 Airbnb 和 Stripe 理解高信任平台：风险识别、支付编排、API、规则、审核、幂等和异常处理决定产品能否规模化。

## 本模块要解决的问题

Airbnb 要让陌生人之间完成住宿交易，必须处理身份、信任、风险、支付、取消和纠纷。Stripe 要让商户安全收款，必须处理 API、支付状态、欺诈、退款、争议、重复请求和 webhook。

## 阅读重点

- 风控不是只有通过或拒绝，还包括验证、审核、限制、拦截、解释和申诉。
- 支付不是一次 API 调用，而是状态、回调、重试、幂等和异常处理的组合。
- API 平台体验包括命名、错误码、版本、认证、分页、限流和 webhook。
- 公开资料能支持 Radar、API、风险 ML 和支付编排方向，但不能披露内部模型权重和策略阈值。
