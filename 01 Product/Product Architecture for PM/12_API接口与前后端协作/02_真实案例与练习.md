# API 练习

## 练习：设计取消订单 API

### 1. 资源

订单：`order`。取消是订单状态变化，不是删除订单。

### 2. 请求需要什么

- 订单 ID。
- 取消原因。
- 幂等键。
- 用户身份和权限。

### 3. 返回需要什么

- 当前订单状态。
- 是否产生退款。
- 下一步提示。
- 如果处理中，如何查询结果。

### 4. 错误要区分

| 错误 | 用户提示 |
|---|---|
| 无权限 | 你不能取消这笔订单 |
| 状态冲突 | 订单已完成，不能取消 |
| 风控限制 | 需要联系客服处理 |
| 系统处理中 | 请求已收到，请稍后查看 |

## 练习：设计支付 webhook

写出：

- 哪些事件会通知：支付成功、失败、需验证、退款。
- webhook 重复到达怎么办。
- 商户系统失败时怎么重试。
- 用户页面如何查询最终状态。

## 自测

如果接口设计没有错误码、幂等、状态查询和权限边界，就还不是可上线设计。
