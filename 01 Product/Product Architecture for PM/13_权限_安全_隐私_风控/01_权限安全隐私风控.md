# 权限、安全、隐私、风控

## 认证和授权别混

认证是确认“你是谁”，授权是确认“你能不能做这件事”。登录成功不等于什么都能操作。

PM 要问：这个功能需要什么角色、什么权限、什么数据范围？

## 隐私不是弹窗

隐私不是加一个同意弹窗，而是数据最小化、用途限制、可解释、可撤回和合规。位置、联系人、支付、健康、儿童数据都要特别小心。

PM 要问：这个数据真的必须收吗？能不能用更少数据达到目标？

## 风控不是只拦坏人

风控要处理异常行为，但也要控制误伤。拦截、审核、验证、限制、申诉是不同动作。

PM 要问：风险分数怎么转成动作？被误伤的正常用户怎么办？

## API 安全和产品体验相关

OWASP API Security Top 10 这类资料提醒我们，API 风险会直接影响产品：越权读取、批量枚举、认证绕过、过度暴露数据，都不是纯技术问题。

PM 要问：用户能否访问别人的数据？列表接口是否可能被批量抓取？错误信息是否泄露敏感内容？

## 评审清单

- 用户是谁，身份如何确认？
- 角色和权限有哪些？
- 数据是否按最小必要原则收集？
- 敏感数据是否需要脱敏、审计、加密或访问限制？
- 风险事件在哪些节点触发？
- 处置动作是否有解释和申诉？
- 安全失败时用户看到什么？

## 常见坑

| 坑 | 后果 |
|---|---|
| 只做前端权限控制 | 容易被绕过 |
| 错把登录当授权 | 用户可能访问不该看的资源 |
| 风控只拦截不申诉 | 误伤正常用户 |
| 为推荐过度收集数据 | 隐私和信任风险 |
