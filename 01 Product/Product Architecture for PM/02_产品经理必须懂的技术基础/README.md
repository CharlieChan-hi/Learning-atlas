# 产品经理必须懂的技术基础

这一模块不是教你写代码，而是教你把常见技术词翻译成产品约束。

## 最小知识地图

| 技术词 | PM 要听懂的意思 |
|---|---|
| 客户端 | 用户手里的 App、网页、设备，负责交互和部分本地状态 |
| 服务端 | 处理业务规则、数据读写、权限、风控和系统协作 |
| 数据库 | 存储核心业务对象和状态 |
| 缓存 | 用空间换速度，但要面对过期和一致性问题 |
| 队列 | 把不必立刻完成的事异步处理 |
| 日志 | 记录发生了什么，用于排查、分析和模型训练 |
| 监控 | 发现系统是否变慢、失败、异常 |
| API | 前后端、系统之间的契约 |

## 学习重点

不要背定义。每个技术词都问三句：

1. 它解决什么产品问题？
2. 它带来什么新风险？
3. PM 评审时应该问什么？
