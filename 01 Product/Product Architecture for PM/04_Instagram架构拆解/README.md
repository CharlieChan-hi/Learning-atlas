# Instagram 架构拆解

本模块用 Instagram 理解内容推荐产品：Feed、Explore、Reels 的核心不是单个模型，而是候选召回、多阶段排序、重排治理和反馈闭环。

## 本模块要解决的问题

用户打开 Instagram 时没有明确搜索词，系统必须判断：哪些内容值得进入候选池，哪些内容优先展示，哪些内容虽然分数高但需要被治理或降权。

产品经理要学会把“推荐更相关”拆成可讨论的问题：内容从哪里来、排序优化什么、负反馈怎么记录、新作者怎么冷启动、低质内容怎么控制。

## 阅读重点

- 召回决定候选边界，排序决定优先级，重排决定体验底线。
- 推荐指标不能只看点击，还要看停留、隐藏、举报、多样性和长期满意度。
- 新作者曝光不是简单加权，要处理质量、安全、兴趣匹配和冷启动。
- Meta 公开资料能支持推荐漏斗和系统思路，但不能支持未披露的当前模型权重或内部服务拓扑。
