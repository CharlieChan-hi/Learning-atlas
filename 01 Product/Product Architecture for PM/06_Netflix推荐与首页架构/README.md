# Netflix 推荐与首页架构

本模块用 Netflix 理解视频首页：推荐系统不只是“选片”，还包括首页行结构、封面图、实验验证和长期内容发现。

## 本模块要解决的问题

用户打开 Netflix 时，真正的问题通常不是内容不够，而是选择成本太高。系统要帮助用户更快开始播放，同时避免只追短期点击、误导性封面、冷启动误判和实验假阳性。

## 阅读重点

- 首页是一组入口和行结构，不是简单推荐 20 部影片。
- 同一内容可以因为不同封面获得不同反馈，展示素材也是推荐的一部分。
- A/B 测试要有主指标和护栏指标，不能只看点击或短期播放。
- Netflix 公开论文和博客能支持方法理解，但不能等同于当前内部完整实现。

## 读完后应该能回答

1. Netflix 首页为什么要降低选择成本？
2. 封面图为什么属于推荐体验的一部分？
3. 新用户冷启动有哪些可用信号和风险？
4. 实验显著但长期留存下降，说明什么？
