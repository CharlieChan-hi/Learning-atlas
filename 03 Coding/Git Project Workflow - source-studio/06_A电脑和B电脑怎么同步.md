> 一句话版本：A 电脑和 B 电脑不是自动同步，必须通过 GitHub 私有仓库作为中间云端中心。

---

## 这一篇解决什么问题

你问：我在 A 电脑用了 Git，B 电脑有没有 Git？

答案是：Git 每台电脑都可以安装，但项目要同步，需要 GitHub 这样的远程仓库。

读完这一篇，你应该能理解 clone、push、pull 的关系。

## 正确模型

两台电脑同步项目，模型是：

```text
A 电脑本地仓库
      ↓ push
GitHub 私有远程仓库
      ↓ clone / pull
B 电脑本地仓库
```

GitHub 在这里像云端中心，但它不是普通网盘。它保存的是 Git 历史。

## 第一次在 B 电脑拿项目

如果 GitHub 上已经有你的私有仓库，B 电脑第一次拿项目用：

```bash
git clone GitHub仓库地址
```

clone 会下载：

- 项目当前文件；
- Git 历史；
- 分支信息；
- 远程仓库连接。

所以 clone 不是普通下载 zip。clone 后，这个文件夹也会是一个 Git 项目。

## A 电脑做完修改

A 电脑修改后，需要：

```bash
git status --short --branch
git diff
git add 相关文件
git commit -m "说明修改原因"
git push
```

其中：

- commit：保存到 A 电脑本地 Git；
- push：上传到 GitHub。

如果 A 电脑只 commit 没 push，B 电脑看不到。

## B 电脑继续工作前

B 电脑开始工作前，应该先：

```bash
git pull
```

pull 的意思是：从 GitHub 拉取别人或另一台电脑已经 push 的最新提交。

如果你忘记 pull，B 电脑可能在旧版本上继续改，后面就容易冲突。

## 最安全的跨电脑习惯

每次换电脑前：

```text
在当前电脑确认已 commit
→ push 到 GitHub
→ 到另一台电脑先 pull
→ 再继续工作
```

每次开始工作前：

```bash
git status --short --branch
git pull
```

前提是你已经有远程仓库。

## 不建议用网盘同步 Git 项目

不要把整个项目，尤其是 `.git` 文件夹，作为主要方式丢到 iCloud Drive、Dropbox、百度网盘里同步。

原因：

- 网盘可能同步到一半；
- `.git` 里有很多内部文件；
- 两台电脑同时改可能互相覆盖；
- 你很难知道哪个历史才是可靠的。

更稳的方式是 GitHub private repository。

## 对你的项目来说

你的 `charliechanstudio.com` 未来应该这样：

```text
Mac A：本地开发
  ↓ push
GitHub private repo：可信源
  ↓ pull / clone
Mac B：继续开发
  ↓ Cloudflare Pages 读取
线上网站部署
```

但现在还没有远程仓库，所以先学会看状态，再建立 private repo。

## 这一篇记住这些

- Git 是每台电脑都可以安装的工具。
- A/B 电脑同步靠 GitHub remote。
- clone 是第一次下载完整 Git 项目。
- push 是上传本机 commit。
- pull 是拿远程最新 commit。
- commit 不等于 B 电脑能看到，push 后才行。
