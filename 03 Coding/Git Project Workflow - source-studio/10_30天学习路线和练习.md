> 一句话版本：30 天学习 Git 的重点不是背命令，而是先用练习仓库建立安全感，再把流程用到真实项目。

---

## 这一篇解决什么问题

你不需要一天学完 Git。Git 是通过反复练习建立感觉的。

读完这一篇，你会有一个 4 周路线：先安全练，再碰真实项目。

## 第 1 周：看懂 Git 状态

目标：不害怕 `git status`。

练习用一个空文件夹，不要用真实网站项目。

步骤：

1. 建一个练习文件夹。
2. 运行：

```bash
git init
```

3. 新建 `hello.md`。
4. 运行：

```bash
git status --short --branch
```

5. 修改 `hello.md`。
6. 运行：

```bash
git diff
```

7. 提交：

```bash
git add hello.md
git commit -m "练习第一次提交"
```

你要学会解释：

- `??` 是什么；
- `M` 是什么；
- add 是什么；
- commit 是什么。

## 第 2 周：看历史和可视化工具

目标：能看到“树根一样”的历史。

练习：

1. 多改几次 `hello.md`。
2. 每次都 commit。
3. 运行：

```bash
git log --oneline --graph --decorate --all
```

4. 安装 GitHub Desktop。
5. 用 GitHub Desktop 打开练习仓库。
6. 对比 CLI 和图形界面看到的信息。

你要理解：每个 commit 都是一个历史点。

## 第 3 周：学习 GitHub 私有仓库

目标：理解本地和云端的区别。

练习：

1. 在 GitHub 创建一个 private repository。
2. 把练习仓库 push 上去。
3. 在另一个文件夹 clone 一份。
4. 在 A 文件夹 commit + push。
5. 在 B 文件夹 pull。

你要学会解释：

```text
commit 为什么不等于 push？
为什么 B 电脑要 pull？
clone 和普通下载 zip 有什么区别？
```

## 第 4 周：分支、回滚、冲突

目标：遇到问题不会乱删。

练习：

1. 创建一个新分支：

```bash
git switch -c practice-branch
```

2. 在新分支改文件并提交。
3. 切回 main：

```bash
git switch main
```

4. 合并分支：

```bash
git merge practice-branch
```

5. 练习取消暂存：

```bash
git restore --staged 文件名
```

6. 练习 revert。

危险命令只了解风险，不在真实项目练习：

```bash
git reset --hard
git clean -fd
git push --force
```

## 什么时候可以碰真实项目

当你能回答这些问题，就可以开始在真实项目上做更正式的 Git 操作：

```text
□ 我能看懂 git status 输出
□ 我知道 M、D、?? 是什么
□ 我知道 commit 和 push 的区别
□ 我知道 remote 是 GitHub 地址
□ 我知道怎么看 diff
□ 我知道 reset --hard 很危险
□ 我知道为什么先用 private repo
```

## 这一篇记住这些

- 先用练习仓库，不要拿真实项目冒险。
- 第 1 周只要看懂 status/diff。
- 第 2 周学历史和可视化。
- 第 3 周学 GitHub 云端。
- 第 4 周学分支和回滚。
- 真实项目永远先看状态再操作。
