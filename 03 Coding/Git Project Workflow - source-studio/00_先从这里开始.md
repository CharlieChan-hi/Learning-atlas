## 这一篇解决什么问题

原来的 Git 文档有 600 多行，内容完整但太长，不适合第一次学习。这一版把它拆成 12 个小模块，每个模块只讲一个主题。

这套资料是给你学习 Git 用的，不是 `charliechanstudio.com` 项目内部文件。它不会修改你的网站项目，也不会执行提交、推送、回滚等 Git 操作。

## 你现在最应该先看哪几个

如果你今天只想开始，不想学太多，先看：

1. `01_Git到底是什么.md`
2. `02_保存_commit_push不是一回事.md`
3. `03_你的项目现在处在什么阶段.md`
4. `04_每天只要先会这几个命令.md`

看完这四个，你就能理解为什么 Git 对你的网站重要，也能看懂最基本的项目状态。

## 不要一上来做的事

在你还没理解之前，不建议对真实的 `charliechanstudio.com` 做这些操作：

```bash
git add .
git commit
git push
git reset --hard
git clean -fd
git push --force
```

这些不是永远不能用，而是你现在还在学习阶段，真实项目里已经有很多未提交改动，乱操作可能造成损失。

## 学习目标

读完这套资料后，你应该能回答：

- Git 是什么？
- commit 和 push 有什么区别？
- GitHub 是不是云端备份？
- A 电脑和 B 电脑怎么同步？
- 我能不能用图形界面看 Git？
- 改坏了怎么回滚？
- main、branch、fork、PR 是什么？
- 我的 `charliechanstudio.com` 应该怎么安全建立远程仓库？
