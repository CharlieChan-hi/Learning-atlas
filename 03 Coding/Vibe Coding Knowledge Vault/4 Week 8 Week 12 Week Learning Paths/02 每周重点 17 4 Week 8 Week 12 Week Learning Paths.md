## 📖 这一节在讲什么

如果你最近就想快速提升，这条路线最适合。重点是“先敢看、敢用、敢判断一点点”。

## 🧭 每周重点

- 第 1 周：`00`、`01` 模块
- 第 2 周：`02`、`03` 模块
- 第 3 周：`04`、`05`、`08` 模块
- 第 4 周：`09`、`10`、`11` 模块

## 🧩 为什么 4 周路线不追求“学完”

因为 4 周的真正目标，不是让你把整套技术体系吃透，而是先建立三样东西：

- 地图感：知道各模块在系统里分别干什么
- 手感：敢打开终端、看代码、看接口、看 Git
- 判断雏形：遇到问题时，知道先从哪一层开始看

如果这三样东西建立起来了，后面继续加深会轻松很多。

## 🎯 每一周更具体应该怎么学

### 第 1 周：先把全景图装进脑子

重点不是死记，而是回答：

- 什么是 Vibe Coding
- PM 为什么现在必须懂更多技术细节
- 一条需求从想法到上线会经过哪些环节

如果这一周读完你开始对“前端、后端、AI、部署”有整体感觉，就算达标。

### 第 2 周：先把终端和 Git 这两个底层抓住

这是整套体系的第一道门槛。你要做到：

- 看见命令不慌
- 看得懂当前目录和路径
- 知道 Git 在管理什么
- 看得懂最基础的改动和分支语义

### 第 3 周：把 Web 和 Python 串起来

这一周最重要的是建立“页面、接口、数据、脚本”的连接感。你会开始感觉自己不是在看零散知识点，而是在看一个系统。

### 第 4 周：把 AI 工具拉进你的工作流

这一周的重点不是学更多模型名，而是学会：

- 如何组织上下文
- 如何要求结构化输出
- 如何让 `Codex CLI` 和 `Claude Code` 帮你做事
- 如何验证 AI 的结果

> 🌐 **In English:**
> The 4-week path is about building orientation, confidence, and basic judgment. It is not about mastering every topic in depth.

## 💡 4 周路线最适合什么人

- 你最近就想快速建立技术手感
- 你已经在工作里开始接触 AI coding 工具
- 你不想被一整本技术书直接压垮
- 你更需要“先上手，再加深”的节奏

## ⚠️ 走 4 周路线时最容易犯的错

### 错误一：读太快，不做最小实践

这样你会以为自己看懂了，但一到终端或代码面前还是会退缩。

### 错误二：每个模块都想深入到底

4 周路线不是深挖路线，它是先打通骨架。

### 错误三：只看 AI，不看基础

这会让你后面很难判断 AI 是否真的在帮你，还是在制造新的混乱。

## 🧪 建议配套练习

你可以每周强制做一个小动作：

- 第 1 周：画出自己的“Vibe Coding 知识地图”
- 第 2 周：在终端里完成 5 个基础动作
- 第 3 周：看懂一个页面从点击到接口再到渲染的流程
- 第 4 周：让 AI 完成一个小修改并自己验证

这四个练习虽然小，但会让 4 周路线真正落地。

## 🎯 本路线的成功标准

- 看懂基础终端输出
- 看懂 Git 状态和简单 diff
- 看懂一个页面和接口闭环
- 能让 AI 帮你完成一个小修改并自己验证
