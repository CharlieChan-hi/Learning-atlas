> 一句话版本：Git 不只能用命令行，你可以用 GitHub Desktop、VS Code、Git Graph、GitLens、GitKraken、SourceTree 看改动和树状历史。

---

## 这一篇解决什么问题

你问 Git 有没有像树根一样的可视化展示。答案是：有。

读完这一篇，你应该知道不同工具适合什么阶段，不会以为 Git 只能在黑色命令行里用。

## Git 可以可视化看什么

图形化工具通常可以看：

- 每次 commit；
- 哪个分支从哪里分出来；
- 哪两个分支在哪里合并；
- 每次改动了哪些文件；
- 当前本机和 GitHub 是否同步；
- 哪些文件还没提交。

## 第一推荐：GitHub Desktop

适合你现在这种还在学习 Git 的阶段。

优点：

- 界面简单；
- 能看到改动文件和 diff；
- 提交、分支、push、pull 都比较直观；
- 和 GitHub 私有仓库配合最自然。

缺点：

- 历史图没有 GitKraken 那么强；
- 太复杂的操作仍然需要理解 Git 原理。

适合用途：第一次创建 GitHub 私有仓库、提交、推送、拉取、看改动。

## 第二推荐：VS Code Source Control

如果你用 VS Code 写项目，左侧 Source Control 面板就能直接看到 Git 状态。

优点：

- 边写代码边看改动；
- 不需要切换太多软件；
- 能查看每个文件的差异；
- 新版 VS Code 也有 Source Control Graph。

缺点：

- 对完全零基础的人，按钮含义仍然需要学习；
- 复杂历史图不一定比专业 Git GUI 清晰。

## 第三推荐：GitLens 或 Git Graph

如果你想在 VS Code 里看到更像“树根/路线图”的历史，可以用 GitLens 或 Git Graph。

你可以这样理解：

```text
VS Code 自带 Git 面板：基础仪表盘
GitLens / Git Graph：更强的历史显微镜
```

适合用途：查看某个文件是谁什么时候改的、查看 commit 图、理解分支历史。

## 第四推荐：GitKraken 或 SourceTree

这类工具更像专业 Git 可视化客户端。

优点：

- 分支树很直观；
- 适合复杂项目；
- 图形化操作多。

缺点：

- 功能多，初学者容易不知道点什么；
- 有些功能可能需要账号或付费；
- 如果不懂 Git 原理，图形按钮也可能误操作。

## 推荐组合

| 阶段 | 推荐工具 | 用途 |
| --- | --- | --- |
| 零基础 | GitHub Desktop | 看改动、commit、push、pull |
| 写代码时 | VS Code Source Control | 边写边看文件状态 |
| 看历史树 | Git Graph / GitLens | 看分支和 commit 图 |
| 复杂项目 | GitKraken / SourceTree | 更完整的图形化 Git 客户端 |
| 精确确认 | CLI 命令 | status、diff、log 最可靠 |

## 我的建议

你先用：

```text
GitHub Desktop + VS Code Source Control
```

等你能看懂 `status`、`diff`、`commit`、`push`、`pull` 后，再用 GitKraken 或 Git Graph 看树状历史。

## 这一篇记住这些

- Git 可以图形化，不只能用 CLI。
- GitHub Desktop 最适合初学者。
- VS Code 适合边写代码边看 Git 状态。
- GitLens / Git Graph 适合看历史树。
- 图形界面不能代替 Git 原理，只是降低操作难度。
