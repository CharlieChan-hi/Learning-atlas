> 一句话版本：HTML 决定页面结构，CSS 决定视觉表现，JavaScript 决定交互行为。

---

## 这一节在讲什么

这一节解释网站前端最基础的三件事：`HTML`、`CSS`、`JavaScript`。你以后看到 Astro、React、Vite，也要知道它们最终仍然服务于这三类结果。

这不是要求你马上写代码，而是让你能分辨一个页面问题大概属于哪一层。

## 概念解释

`HTML` 是页面结构。标题、段落、链接、按钮、图片、列表，这些都需要结构。

`CSS` 是页面外观。字体大小、颜色、间距、布局、边框、响应式变化，都属于视觉表现。

`JavaScript` 是页面行为。点击按钮后发生什么、数据如何更新、某个组件如何响应用户操作，通常与 JavaScript 有关。

现代框架会把它们包装起来。例如 Astro 文件里可能同时出现页面结构、导入组件和脚本逻辑。React 组件也可能把结构和交互写在一起。

## 现实例子

一个按钮写着「View projects」。按钮文字和链接是结构与内容。

按钮是黑色背景、白色文字、有圆角和留白，这是 CSS。

如果点击按钮后弹出窗口、切换状态或加载数据，这就是 JavaScript 的工作。

## Charlie 项目案例

Charlie 首页 `apps/web/src/pages/index.astro` 里有 `<h1>`、`<p>`、`<a>` 等结构，它们对应页面内容和链接。

全局样式在 `apps/web/src/styles/global.css`，它控制视觉风格。

项目卡片组件 `apps/web/src/components/ProjectCard.astro` 把项目标题、摘要和标签组合成可点击卡片。

这说明真实项目里，HTML、CSS、JavaScript 的边界会被框架组织起来，但底层概念仍然存在。

## 小练习

1. 找一个网页按钮，判断它的文字属于哪一层。
2. 判断它的颜色和间距属于哪一层。
3. 判断点击后的行为属于哪一层。

## 检查问题

1. `HTML` 主要解决页面长相，还是页面结构？
2. `CSS` 是否负责 DNS？
3. Astro 和 React 是否会让 HTML、CSS、JavaScript 的基础概念消失？

## 常见误解

新手常把所有前端问题都叫「代码问题」。更准确的做法是先判断问题属于结构、表现还是行为。

另一个误解是认为框架比基础更重要。框架会变，但 HTML、CSS、JavaScript 的分工长期稳定。

## 记住这些

- HTML 是结构。
- CSS 是表现。
- JavaScript 是行为。
- 框架会组织它们，但不会替代它们。
