> 一句话版本：浏览器负责访问和显示页面，服务器负责提供资源，domain 和 DNS 负责把人能读的地址指向正确位置。

---

## 这一节在讲什么

这一节解释浏览器、服务器、domain 和 DNS 的关系。你不需要马上配置它们，但要先知道它们分别解决什么问题。

如果你未来要把个人网站上线到 Cloudflare Pages，这些概念会反复出现。

## 概念解释

浏览器是你打开网站的工具，例如 Chrome、Safari、Edge。它会根据网址请求页面资源，然后把页面显示出来。

服务器是提供资源的地方。对静态网站来说，服务器通常把构建好的 `HTML`、`CSS`、`JavaScript`、图片等文件发给浏览器。

`domain` 是人能读的地址，例如 `charliechanstudio.com`。它比一串机器地址更容易记住。

`DNS` 像地址查询系统。浏览器知道 domain 后，需要通过 DNS 找到这个 domain 对应的服务位置。

## 现实例子

你在浏览器输入 `charliechanstudio.com`。浏览器先问 DNS，这个 domain 应该去哪里找。

DNS 返回对应位置后，浏览器向网站服务请求页面。服务返回页面资源，浏览器再显示首页。

如果 DNS 配错，网站代码可能没问题，但访问不到。如果构建输出错了，DNS 可能没问题，但页面显示不对。

## Charlie 项目案例

只读观察显示，Charlie 项目的 Astro 配置里有：

```js
site: "https://charliechanstudio.com"
```

这个配置告诉 Astro，这个站点的正式网址是什么。它会影响 SEO、链接生成和站点元信息。

学习时你要知道，`site` 配置不是页面文案，它属于网站上线和地址体系的一部分。

## 检查问题

1. DNS 负责设计页面吗？
2. 浏览器显示页面前，需要先拿到什么？
3. `site` 配置为什么和正式网址有关？

## 常见误解

常见误解是把 domain 当成网站文件本身。domain 只是地址，不是页面内容。

另一个误解是认为网站打不开一定是代码错。实际可能是 DNS、部署、缓存、构建输出或访问路径问题。

## 记住这些

- 浏览器负责访问和显示。
- 服务器或托管平台负责提供资源。
- domain 和 DNS 负责让地址找到正确位置。
