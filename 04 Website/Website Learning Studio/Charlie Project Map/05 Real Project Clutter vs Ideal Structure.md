## 这一节在讲什么

这一节解释真实项目为什么看起来会比教材复杂。你会学习如何区分理想结构和现实结构，避免被临时文件、历史目录或构建输出干扰。

这是非常重要的新手能力。真实项目很少像教程一样干净。

## 概念解释

理想结构是教学时画出来的清爽模型。它帮助你理解职责分工。

真实项目会包含更多东西，例如 `.git`、`node_modules`、`dist`、备份目录、旧文件、测试文件、文档、临时输出。

学习时要先抓主线。主线通常包括根配置、应用入口、内容目录、共享包和部署配置。

旁边噪音不一定没用，但不适合一开始读。

## 现实例子

你整理书桌时，教材会告诉你书放一边、笔放一边、文件夹放一边。

真实书桌上可能还有收据、草稿、旧笔记、临时便签。它们不代表你的学习路线要从收据开始。

项目目录也是这样。看到复杂内容时，先问：它是不是当前理解网站运行的主线？

## Charlie 项目案例

只读观察显示，Charlie 项目里有 `node_modules`，也有 `.node_modules.broken-20260507231316` 这样的历史或修复相关目录。

还有 `apps/web/dist 2/`、`apps/focus-training/dist 2/` 等未跟踪目录出现在 Git 状态里。这些很可能是已有的本地输出或副本，不应在本学习任务中处理。

当前学习主线应放在：

- `package.json`
- `pnpm-workspace.yaml`
- `apps/web`
- `packages/content`
- `packages/lib`
- `packages/ui`
- `content/projects`
- `cloudflare`

本次任务不会清理、移动或修改 Charlie 项目里的任何文件。

## 小练习

1. 把真实项目目录分成「主线」和「暂时不读」两类。
2. 解释为什么新手不应该先读 `node_modules`。
3. 写下你看到复杂目录时的第一句自我提醒。

## 检查问题

1. 真实项目里出现临时目录，是否等于项目不能学习？
2. `node_modules` 是学习业务代码的第一入口吗？
3. 为什么本学习库要记录「只读确认」和「待只读确认」？

## 常见误解

常见误解是把真实项目里的所有文件都当成同等重要。学习有顺序，主线优先。

另一个误解是看到未提交改动就想立刻清理。除非任务明确要求，否则不要碰不属于本任务的改动。

## 记住这些

- 真实项目会比教材乱。
- 先读主线，不要被临时目录带偏。
- 不属于本任务的改动不要处理。
