> 一句话版本：Terminal 是用文字命令操作电脑和项目的入口，文件路径是命令找到目标的地址。

---

## 这一节在讲什么

这一节解释 terminal、files 和 path。你会学习为什么开发项目经常用命令行，以及如何理解文件路径。

你不需要记住所有命令，但要知道命令是在什么位置、对什么文件执行。

## 概念解释

`Terminal` 是命令行工具。你可以用文字命令列出文件、进入目录、运行脚本、检查状态。

文件路径是文件的位置。绝对路径从根开始，例如 `/Users/mac/Desktop/Website Learning Studio`。相对路径从当前所在目录开始。

当前目录很重要。同一个命令在不同目录执行，结果可能不同。

带空格的路径需要加引号，例如 `"Website Learning Studio"`，否则命令可能把它当成多个部分。

## 现实例子

如果你在 Finder 里双击文件，是用图形界面打开。

如果你在 Terminal 里输入命令，是用文字方式操作。

两者都在操作文件，只是方式不同。学习网站时，命令行常用于安装依赖、运行项目、构建和检查。

## Charlie 项目案例

Charlie 项目路径是 `/Users/mac/Desktop/charliechanstudio.com`。如果要查看它的 Git 状态，可以使用：

```bash
git -C "/Users/mac/Desktop/charliechanstudio.com" status --short
```

这里的 `-C` 表示让 Git 在指定目录执行，不需要先 `cd` 进去。

本学习库只读使用过这个命令来确认 Charlie 项目的既有改动。

## 常见误解

常见误解是认为命令行很危险。命令行本身只是工具，危险来自不理解命令含义就执行破坏性操作。

另一个误解是以为所有命令都能在任意目录运行。很多项目命令需要在项目根目录或指定路径运行。

## 记住这些

- Terminal 用文字操作项目。
- 路径是文件地址。
- 执行命令前先确认目标目录。
