> 一句话版本：未来问题清单用来保存暂时不解决的问题，让学习不中断也不丢线索。

---

## 这一节在讲什么

这一节提供 future questions 的使用方式。你会学会把不懂的问题放进一个稳定位置，之后再回来处理。

零基础学习最怕被一个术语卡住。问题清单可以保护你的学习节奏。

## 概念解释

future question 是暂时不深入的问题。它可能是一个术语、一个路径、一个命令、一个设计判断或一个部署疑问。

记录问题时要写清楚来源。最好包含文件路径、文章名或你看到的原句。

问题可以分成四类：概念问题、项目问题、设计问题、操作问题。

## 问题模板

```text
问题：
来源：
我现在的理解：
我卡住的点：
下一次要查什么：
```

## 现实例子

问题：`slug` 是什么？

来源：`packages/content/src/projects.ts` 和 `content/projects/focus-training.en.mdx`。

我现在的理解：它像项目的短名字，会进入 URL。

我卡住的点：它和文件名、frontmatter 的关系还不熟。

下一次要查什么：读 `schema.ts` 和项目详情页 route。

## Charlie 项目案例

可以先记录这些未来问题：

1. `apps/console` 的具体用途是什么？
2. `cloudflare` 目录里有哪些部署或函数配置？
3. `packages/ui` 里哪些组件会影响项目展示？
4. 首页未来如果强化个人介绍，需要改哪些文件？
5. Cloudflare Pages 的 build command 和 output directory 应该如何设置？

这些问题不需要在第一天全部解决。

## 小练习

1. 写下你现在最不懂的 5 个问题。
2. 给每个问题分类。
3. 选一个最值得下次解决的问题。

## 检查问题

1. 遇到不懂问题时是否必须马上查完？
2. 记录来源为什么重要？
3. 未来问题清单如何帮助长期学习？

## 常见误解

常见误解是问题越少代表学得越好。真实学习中，问题会变得更具体，这反而是进步。

另一个误解是把问题清单当成压力清单。它只是存放线索，不是当天必须完成的任务。

## 记住这些

- 问题先记录，不要打断主线。
- 写清来源，方便回查。
- 问题变具体就是进步。
