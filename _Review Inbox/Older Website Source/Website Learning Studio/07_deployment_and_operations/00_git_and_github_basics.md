## 这一节在讲什么

这一节解释 Git 和 GitHub。你会学习为什么网站项目要记录版本，以及为什么部署平台常和 GitHub 连接。

本节不是 Git 命令大全，而是帮助你理解版本管理的意义。

## 概念解释

`Git` 是版本控制系统。它记录文件变化，让你知道改了什么，也能在需要时回到旧版本。

`GitHub` 是代码托管平台。它可以保存 Git 仓库、展示提交历史、管理协作和连接部署平台。

`commit` 是一次变化记录。`branch` 是一条开发线。`status` 可以查看当前工作区有哪些变化。

## 现实例子

如果你修改首页文案，Git 可以记录这次修改。

如果修改后发现效果不好，可以查看差异，决定是否恢复。

如果多人一起做网站，GitHub 可以让大家看到变更历史，而不是互相传压缩包。

## Charlie 项目案例

只读检查 Charlie 项目的 Git 状态时，发现已有多个未提交改动和未跟踪文件。这些改动在本学习任务之前已经存在。

本任务不会修改 Charlie 项目，因此不应该清理、提交或覆盖这些改动。

这个案例说明，开始任何任务前先看 `git status` 很重要。它能告诉你工作区是否已经有别人或之前留下的变化。

## 常见误解

常见误解是 Git 只给程序员使用。只要你维护网站，版本记录就很有价值。

另一个误解是看到未提交文件就要删除。那可能是别人的工作或重要临时产物。

## 记住这些

- Git 记录变化。
- GitHub 托管仓库和协作历史。
- 修改前先看状态，避免覆盖别人的工作。
