# Website Learning Studio

这个资料库是给零基础学习者使用的个人网站学习库。它不负责改正式网站，也不负责替你做最终设计；它负责把网站的概念、目录、代码、内容、设计和上线流程拆成可以逐步理解的课程。

你读它时可以只抓一条主线：先看懂网页在表达什么，再看懂项目文件如何支撑这个表达，最后再理解它如何上线。不要一开始就把目标设成“我要会写所有代码”。更适合的目标是：我能判断一个个人网站为什么这样设计，也能知道要改哪里、问什么、检查什么。

## 这套学习库真正解决什么问题

它解决的是一个小白最容易卡住的问题：看到网站时只看到表面，看到项目时只看到一堆文件，想设计网页时又不知道从哪里开始。

所以这套学习库把学习拆成三条线。

第一条线是网页理解线。你会知道浏览器、页面、内容、样式、组件、路由、域名和部署之间的关系。

第二条线是个人网站设计线。你会知道首页第一屏为什么要先讲“我是谁、我做什么、为什么值得继续看”，项目为什么适合放在下方，文字、留白、层级和信任感如何一起工作。

第三条线是 Charlie 项目地图线。你会把真实路径和概念连起来，例如 `content/projects` 是内容来源，`packages/content` 负责读取内容，`apps/web/src/pages` 负责页面入口，`components` 负责显示结构。

## 你会在这里学到什么

你会先理解网站是什么，再理解一个真实项目为什么会有这么多目录。然后你会学习个人介绍型网站的设计逻辑，知道为什么首页第一屏要先回答「我是谁、我做什么、为什么值得继续看」，而不是一打开就堆项目。

这个资料库会把专业术语保留下来，例如 `HTML`、`CSS`、`JavaScript`、`component`、`route`、`build command`、`Cloudflare Pages`。术语不会被跳过，每个重要词都会在上下文里解释。

读到术语时，你不需要立刻背下来。你只需要先知道它属于哪一层：页面结构、视觉表现、内容数据、构建工具，还是上线访问。

## 30 分钟第一次阅读路线

第一轮不要追求全部记住，只建立整体感觉。

前 10 分钟读 `00_orientation`。你只需要知道自己现在在哪一层，不需要立刻学会写代码。

中间 10 分钟读 `01_website_big_picture`。重点看浏览器、服务器、域名、HTML、CSS、JavaScript 的关系。

最后 10 分钟读 `03_personal_site_strategy` 的第一篇。你会知道个人网站不是作品堆放页，而是读者第一次认识你的入口。

如果读到一半觉得抽象，就停下来问一个问题：这个知识是在帮我看懂页面、看懂文件，还是帮我设计首页？只要能归类，就说明你没有迷路。

## 设计网页时先想什么

个人介绍型网站最先要想的不是颜色，也不是动效，而是读者打开页面后的理解顺序。

一个小白可以按这个顺序设计首页：

1. 写出一句身份定位，例如“我是谁，我关注什么”。
2. 写出两三行补充说明，例如“我为什么做这个网站，读者能看到什么”。
3. 选择两个主要入口，例如“查看项目”“了解经历”。
4. 放入一段可信线索，例如学习背景、长期方向、代表项目或方法。
5. 再往下放精选项目，让项目证明前面的介绍。

这个顺序比先找模板更稳。模板只能给你形状，不能替你决定表达。

## 这个资料库不做什么

它不会修改 `/Users/mac/Desktop/charliechanstudio.com`。

它不会启动 dev server，不会运行正式网站，也不会替你把设计定稿。

它不会把你变成一夜之间能独立写完整网站的工程师。它的目标更实际，让你能看懂项目、提出判断、知道下一步该问什么。

## 每篇文章怎么读

大多数文章都采用同一种结构：

- 一句话版本
- 这一节在讲什么
- 概念解释
- 现实例子
- Charlie 项目案例
- 小练习
- 检查问题
- 常见误解
- 记住这些

如果你读到某个词不懂，先不要停太久。把它记到 `09_reference/future_questions.md`，继续往下读。学习网站最怕一开始就卡在一个术语上。

## 读不进去时怎么办

如果你觉得某篇文章读不进去，不要直接判定自己不适合学网站。通常只是因为你还不知道这篇文章解决哪类问题。

可以换成三个小问题：

1. 这篇在讲页面表面、项目文件、设计判断，还是上线流程？
2. 我能不能从文中摘出一个真实路径、一个术语、一个练习？
3. 读完后我能不能用 30 秒讲给别人听？

如果能回答其中一个，就继续。学习网站靠的是逐步建立地图，不是一次读完全部。

## 质量标准

好的学习资料应该让你能够复述，而不是只觉得「好像懂了」。所以每篇文章都要给你一个小练习和自测问题。

如果一篇文章只讲概念，没有现实例子，它不够好。如果只讲操作，没有解释为什么，它也不够好。这个资料库追求理论和实践参半。
