> 📌 **一句话版本：** 这是一套面向产品经理的 Vibe Coding 系统知识库，目标不是把你变成“科班程序员”，而是让你具备理解代码、与 AI 协作、判断技术方案、推动产品落地的完整能力。

---

## 📖 这一套资料在讲什么

这套资料把 `Vibe Coding` 拆成了三层能力：

1. 你需要知道技术世界是怎么运转的，至少知道前端、后端、数据库、AI、部署分别在干什么。
2. 你需要会和代码打交道，能看懂报错、读懂结构、提出靠谱的改动要求，而不是只会写需求文档。
3. 你需要掌握和 AI 协作的工作流，包括 `Codex CLI`、`Claude Code`、终端、Git、MCP、调试、验证与交付。

> 🌐 **In English:**
> This knowledge base is designed for product managers who want to work effectively with code and AI coding tools. It focuses on understanding systems, making sound technical judgments, and shipping real outcomes with modern agentic workflows.

## 🧭 适合谁

- 想补齐技术认知的产品经理
- 想把 AI 用到真实研发协作里的非工程背景同学
- 想从“会提需求”升级到“能判断方案、能读代码、能带着 AI 推进落地”的学习者

## 🗂 模块总览

- `00_导航与学习方法`：告诉你这套资料怎么读、怎么练、怎么建立学习节奏。
- `01_Vibe_Coding全景与认知升级`：先建立全景地图，避免只学零散工具。
- `02_终端_Shell_文件系统与环境`：解决“看到命令行就慌”的问题，并补齐 `JSON`、配置文件和环境管理。
- `03_Git_GitHub与协作流`：让你真正看懂版本控制和团队协作。
- `04_Web基础_HTML_CSS_JS_TS`：理解前端世界的核心分工。
- `05_前端工程化_React_Next与接口联调`：理解现代前端项目为什么复杂。
- `06_后端基础_HTTP_API_鉴权与服务`：理解服务端和接口的核心逻辑，以及同步、异步、并发与任务系统。
- `07_数据库_SQL_NoSQL_缓存与数据建模`：理解数据如何存、查、改、控。
- `08_Python基础与自动化`：掌握最适合 PM 上手的脚本能力，包括脚本自动化与基础并发。
- `09_AI基础_LLM_RAG_Agent_MCP_Eval`：补齐 AI 应用的底层认知，理解结构化输出、工具调用与工作流编排。
- `10_AI_Coding工具_Codex_CLI_Claude_Code`：学习怎样把 AI 变成真正的工程搭子。
- `11_产品经理的技术判断力`：培养你对需求、方案、报错和代码质量的判断。
- `12_调试_测试_代码质量与Review`：建立最容易被忽视但最关键的质量能力。
- `13_部署运维_Docker_CI_CD_监控回滚`：理解“上线”到底意味着什么。
- `14_安全_权限_密钥_成本与风控`：避免最常见也最致命的工程风险。
- `15_完整实战案例_从PRD到上线`：把前面的知识串成闭环。
- `16_术语表_命令速查_报错速查`：解决学习和实战时的速查问题，特别适合查 `JSON`、命令、报错和接口字段。
- `17_4周_8周_12周学习路线`：提供不同投入强度的训练路径。

## 🧪 如何使用这套资料

- 不要顺着文件名机械阅读，先按路线读，再回到模块内部精读。
- 每学完一个概念，都要在终端或代码仓里做一次最小实践。
- 每看见一个英文术语，不要死记，先建立“它解决什么问题”的感觉。
- 每周至少做一次“从需求到改动建议”的小练习，逼自己把知识串起来。
