> 📌 **一句话版本：** 真正稳定的后端，不只是“能成功处理”，还要能优雅地失败、合理地重试，并避免重复执行带来的副作用，这就是错误处理、幂等和限流的重要性。

---

## 📖 这一节在讲什么

很多后端设计难点都不在正常路径，而在“出错时怎么办”。这页把几个高频但特别容易忽略的稳定性概念放在一起讲清楚。

## 🧠 什么是错误处理

错误处理不是简单返回一句“失败了”，而是要考虑：

- 错在输入、权限、业务条件，还是系统内部
- 前端和用户是否能理解
- 后续能不能定位和复现

## 🧠 什么是重试

当某些失败是暂时性的，例如网络抖动、外部服务偶发超时，系统可能会尝试再次执行。

## 🧠 什么是幂等

幂等的意思是：同一个请求即使重复执行，也不应该造成重复副作用。

这个概念在：

- 支付
- 提交表单
- 异步任务
- AI 高成本调用

里都非常重要。

## 🧠 什么是限流

当请求量过大时，系统需要限制流量，避免被打崩或被异常调用拖垮。

> 🌐 **In English:**
> Reliable backends need more than success flows. They need clear error handling, safe retries, idempotency, and rate limiting to stay stable.

## ⚠️ PM 为什么也必须知道这些词

因为它们会直接影响：

- 用户重复点击后会不会生成两份结果
- 系统失败后能不能自动恢复
- 高成本功能会不会被滥用
- 某些接口为什么不能无限重试

## 🧪 一个很有用的提问方式

遇到重要接口时，你可以问：

1. 失败时怎么提示
2. 会不会自动重试
3. 重复请求会不会重复创建
4. 有没有限流或配额

只要这四问答不清，这个接口的稳定性设计大概率还不完整。

## 📝 记住这些

- 后端稳定性不是靠“别出错”，而是靠“出错时也有设计”。
- 幂等和限流看起来很技术，但本质上都和产品体验、成本和风险直接相关。
- PM 越理解失败路径，越能推动系统变稳。
