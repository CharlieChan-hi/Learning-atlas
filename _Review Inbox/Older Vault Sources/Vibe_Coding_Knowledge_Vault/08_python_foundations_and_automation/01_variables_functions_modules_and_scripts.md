> 📌 **一句话版本：** 变量、函数、模块和脚本，是 Python 最基础也最实用的四块地基；你只要先把它们理解透，后面大多数小自动化都能上手。

---

## 📖 这一节在讲什么

这页不打算把 Python 语法大全塞给你，而是只抓对 PM 最实用的四个概念，让你能看懂脚本的基本骨架。

## 🧠 变量

变量可以理解成给一段数据起个名字，方便后面反复使用。

## 🧠 函数

函数可以理解成把一段重复逻辑封装起来，想用时再调用。

## 🧠 模块

模块是 Python 组织代码和复用能力的一种方式。你可以把常用功能放在一个文件或库里，再在别处导入使用。

## 🧠 脚本

脚本就是一段可以直接执行的 Python 程序，通常用来完成一个具体任务。

> 🌐 **In English:**
> Variables store data, functions package repeatable logic, modules organize code, and scripts turn those pieces into executable tasks.

## 🎯 为什么这四个概念最关键

因为你以后看到的大多数 Python 小工具，本质上都长这样：

- 定义一些变量
- 写几个函数
- 导入几个模块
- 组成一个脚本流程

## 🧩 这四个概念怎么在一个小脚本里配合

你可以把一个典型脚本想成：

- 先用变量接住输入
- 再写函数处理逻辑
- 再导入模块完成文件或接口操作
- 最后在脚本入口把流程串起来

这也是为什么你不需要一开始看很复杂的框架，就已经能理解很多真实脚本。

## 🎯 变量最值得理解的不是语法，而是“状态承载”

变量经常承担的是：

- 用户输入
- 接口返回
- 文件内容
- 处理中间结果

只要你知道一个变量现在装的是什么，很多脚本逻辑就容易看懂。

## 🎯 函数最值得理解的是“边界”

一个函数通常应该解决一件相对清晰的事，例如：

- 读取文件
- 调一个接口
- 解析一段 JSON
- 格式化输出

当函数边界清楚时，脚本更容易维护，也更容易让 AI 帮你局部改。

> 🌐 **In English:**
> Variables hold state, functions isolate responsibilities, modules provide reusable capability, and scripts connect them into an executable flow.

## 💡 模块为什么对 PM 也重要

因为很多时候你并不是自己从零写功能，而是在看：

- 这个脚本从哪个库里导了什么
- 这个函数是不是来自标准库还是第三方库
- 这段能力是本地文件里的，还是外部依赖里的

一旦你理解模块，你读 Python 代码的“迷路感”会下降很多。

## ⚠️ 最容易出现的误区

### 误区一：把整个脚本写成一大坨

这样短期能跑，后面很难改、很难测、也很难让 AI 帮你局部处理。

### 误区二：看到导入就慌

其实导入只是说明“这段能力来自哪里”，不用一上来全都掌握。

### 误区三：只看语法，不看数据流

真正决定你能不能看懂脚本的，往往不是语法，而是“数据从哪里来，经过什么处理，最后去哪了”。

## 🧪 一个适合练习的小脚本骨架

你完全可以先练这种结构：

1. 定义输入变量
2. 写一个读取函数
3. 写一个处理函数
4. 写一个输出函数
5. 在底部把它们连起来

这会比上来就啃复杂工程代码友好得多。

## 📝 记住这些

- 不必一开始追求全面语法，只要先理解脚本的结构。
- 看懂变量、函数、模块和脚本，你就能开始读和改很多实用代码。
- AI 帮你写脚本时，这四个概念会成为你最重要的理解锚点。
