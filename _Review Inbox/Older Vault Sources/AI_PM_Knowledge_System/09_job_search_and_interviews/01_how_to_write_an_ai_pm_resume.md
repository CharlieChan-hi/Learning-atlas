📌 **一句话版本**
AI PM简历的核心不是"堆AI关键词"，而是用具体的AI使用场景和量化数据，证明你有能力将AI技术转化为产品价值。

> 🌐 **In English:**
> An AI PM resume should not keyword-stuff. It must demonstrate, with specific examples and metrics, your ability to translate AI capabilities into product value.

## 📖 这一节在讲什么

本节解决一个传统PM最纠结的问题：**我没做过AI产品，简历该怎么写？**

我们会对比传统PM简历和AI PM简历的本质差异、梳理如何用已有经验"翻译"成AI PM语言、提供可直接使用的简历结构模板，并列出最常见的错误。

## 🧠 一、传统PM简历 vs AI PM简历的区别

### 1.1 核心差异对比

| 维度 | 传统PM简历 | AI PM简历 |
|------|-----------|-----------|
| 技术关键词 | 敏捷开发/需求文档 | LLM/RAG/Prompt工程 |
| 量化指标 | DAU/GMV/留存率 | 模型准确率/幻觉率/延迟 |
| 项目描述重点 | 功能上线 vs 用户价值 | AI能力边界+产品策略 |
| 技术理解证明 | 一般不需要 | 需要展示AI工具使用经验 |

### 1.2 思维方式的转变

传统PM简历思维：**"我做了什么功能，带来了什么结果"**

AI PM简历思维：**"我如何判断AI在这个场景的适用性，如何设计人机协作流程，如何评估AI效果"**

这个思维差异是招聘方筛选简历时最先感知到的。

> 🌐 **In English:**
> Traditional PM resume: "what feature I built and the business result."
> AI PM resume: "how I assessed AI applicability, designed the human-AI workflow, and measured AI quality."

## 🧠 二、如何突出AI经验（即使没有直接经验）

### 2.1 三种经验转化策略

**策略一：工具使用经验**

把你使用AI工具的深度经验写进去。

```
❌ 弱版：熟悉ChatGPT、Midjourney等AI工具

✅ 强版：
- 使用Claude API构建内部知识库问答系统
  原型，通过RAG技术降低幻觉率约40%
- 系统评测过GPT-4o/Claude3/Gemini在
  客服场景的准确率和响应速度
```

**策略二：旧项目AI视角重述**

把你之前做的非AI项目，用AI PM的视角重新描述。

```
❌ 旧版：
负责用户智能推荐系统改版，提升CTR 15%

✅ 新版：
主导用户推荐系统从规则引擎到ML模型的迁移：
- 定义模型评估指标（AUC/Precision/Recall）
- 设计A/B实验框架，验证模型上线效果
- CTR提升15%，召回率提升23%
```

**策略三：副项目/个人项目**

如果没有工作中的AI经验，用个人项目填补：

- 用Coze/Dify搭建一个AI应用并写分析报告
- 对某款AI产品做完整的竞品分析和改进方案
- 参与开源AI项目的产品文档撰写

### 2.2 快速积累AI经验的方法

```
本周可以做的：
1. 注册Claude/GPT API，用Python跑通
   一个最简单的对话接口
2. 用LangChain/Dify搭建一个简单的
   RAG知识库
3. 写一篇500字的AI产品使用体验报告

本月可以做的：
1. 完成一份完整的AI产品竞品分析
2. 设计一个AI功能的PRD
3. 参加一次AI Hackathon
```

> 🌐 **In English:**
> Three strategies: (1) document your AI tool usage with depth and metrics, (2) reframe past work with AI/ML language, (3) build side projects to fill gaps. Start this week.

## 🧠 三、关键词优化策略

### 3.1 必须出现的AI关键词

按重要性排列：

**第一梯队（核心关键词）：**
- LLM / 大语言模型
- Prompt Engineering / 提示词工程
- RAG（检索增强生成）
- AI产品评估 / 效果评测
- 人机协作设计

**第二梯队（加分关键词）：**
- Fine-tuning / 模型微调
- RLHF / 对齐
- AI Agent / 智能体
- Embedding / 向量数据库
- 幻觉（Hallucination）控制

**第三梯队（行业通用词）：**
- 生成式AI / AIGC
- 多模态
- 对话式UI
- AI伦理与安全

### 3.2 关键词使用原则

⚠️ **不要堆砌关键词。** 每一个关键词出现，都必须有对应的具体场景支撑。

```
❌ 错误：熟悉RAG、Fine-tuning、RLHF、向量数据库

✅ 正确：
- 主导知识库产品RAG架构设计，通过
  chunk策略优化使检索准确率提升28%
- 参与RLHF数据标注规范制定，负责
  1000+条对话偏好数据的质量审核
```

## 🧠 四、项目经历怎么写：STAR法则 + AI适配

### 4.1 传统STAR vs AI-STAR

传统STAR：Situation（背景）/ Task（任务）/ Action（行动）/ Result（结果）

AI-STAR新增两个维度：
- **AI Fit（AI适配度判断）：** 为什么这个场景适合用AI？
- **Limitation（边界认知）：** 遇到了哪些AI的局限，如何处理？

这两个维度是AI PM区别于普通PM最核心的思维体现。

### 4.2 项目经历模板

```
【项目名称】AI客服质检系统
【时间规模】2024.03-2024.09 | 独立负责

S（背景）：
传统人工质检效率低，覆盖率不足5%，
无法支撑千万级对话量的质量管控需求。

T（任务）：
设计并落地基于LLM的自动化质检产品，
将质检覆盖率提升至80%以上。

A（行动）：
- 评估GPT-4o/Claude3用于质检场景的
  准确率和成本，最终选择Claude-3-Haiku
- 设计6个维度的质检Rubric，并迭代
  Prompt 4个版本
- 设计人工复核兜底机制，处理AI误判

AI Fit（适配判断）：
质检本质是语义理解+规则判断，LLM
在这类结构化评估场景中准确率稳定，
且边际成本极低，适合大规模部署。

Limitation（边界）：
对情绪识别类指标，AI判断一致性较差，
通过人工标注200条边界案例做Few-shot
优化，将该指标准确率从62%提升至79%。

R（结果）：
- 质检覆盖率从5%提升至83%
- 质检人力成本降低65%
- 发现客服问题响应速度提升40%
```

### 4.3 量化指标选取指南

| 类型 | 传统指标 | AI项目指标 |
|------|---------|-----------|
| 质量 | 用户满意度 | 模型准确率/F1分数 |
| 效率 | 开发周期缩短 | 推理延迟/Token成本 |
| 业务 | GMV/DAU | 自动化率/人工替代率 |
| 安全 | 系统稳定性 | 幻觉率/违规率 |

> 🌐 **In English:**
> Add two dimensions to the STAR framework: AI Fit (why this use case suits AI) and Limitation (what AI can't do and how you handled it). These reveal genuine AI PM thinking.

## 🧠 五、简历模板结构

### 5.1 标准AI PM简历结构（一页）

```
[姓名] | AI产品经理
[邮箱] | [手机] | [LinkedIn] | [作品集链接]

── 核心技能 ──────────────────────
AI能力：LLM产品设计 | Prompt工程 |
        RAG架构理解 | AI评测方法论
产品能力：PRD/用户研究/A-B测试/数据分析
工具：Claude API / LangChain / Dify /
      Notion / Figma / SQL

── 工作经历 ──────────────────────
[公司名] - AI产品经理      2023.06-至今
项目：[AI产品名]
• [用AI-STAR模板写，约3-4个bullet]
• 每条bullet必须有量化数据

[公司名] - 产品经理        2021.03-2023.05
项目：[产品名]（用AI视角重述）
• [重点写AI/数据/算法相关经验]
• [其余经验可简写]

── AI项目（个人/副业） ─────────────
[项目名]（2024.02）
• 基于RAG构建的[场景]工具
• 主要技术：[工具栈]
• 效果：[量化结果]

── 教育背景 ──────────────────────
[学校] [专业] [学历] [毕业年份]

── 认证 ──────────────────────────
DeepLearning.AI: Prompt Engineering
Google: AI Essentials（可选）
```

### 5.2 关键格式规则

- **长度：** 严格控制在1页（工作经验5年以内）
- **字体：** 中文简历用思源黑体或苹方，英文简历用Calibri
- **链接：** 作品集链接必须放在名字旁边，且可以直接点击
- **文件名：** `姓名_AI产品经理_2025.pdf`

## 🧠 六、常见错误清单

⚠️ **以下是最常见的简历错误，对照检查：**

```
❌ 错误1：只写职责，不写结果
   "负责AI功能的需求对接和跟进"
   → 改为：主导AI客服功能上线，降低
     人工处理量35%

❌ 错误2：AI关键词堆砌无场景
   "熟悉ChatGPT/Claude/Gemini/DALL-E"
   → 改为：使用Claude API设计[场景]产品
     原型，处理[数量]次对话

❌ 错误3：技术描述不准确
   "使用深度学习做推荐系统"
   → 如果你不确定技术细节，
     描述你的职责角色，而非技术实现

❌ 错误4：没有作品集链接
   AI PM简历必须附作品集，
   这是传统PM最大的差异点

❌ 错误5：格式混乱，PDF解析失败
   用Overleaf/标准Word模板
   导出PDF，避免排版错位
```

> 🌐 **In English:**
> Top 5 resume mistakes: (1) duties without results, (2) keyword stuffing without context, (3) inaccurate tech descriptions, (4) no portfolio link, (5) broken PDF formatting.

## 🎯 AI产品经理视角

简历是一个**信号筛选机制**，招聘方在30秒内判断你是否值得约面试。

AI PM招聘方看简历的顺序：
1. 有没有AI产品的直接经历？
2. 量化数据是否可信？
3. 技术关键词是否与岗位JD匹配？
4. 有没有作品集链接？
5. 项目规模是否与声称的职级匹配？

所以简历优化的优先级是：**真实经验 > 量化数据 > 关键词匹配 > 格式美观**。

## 📝 记住这些

- AI PM简历 = 传统PM经验 + AI场景翻译 + 量化数据支撑
- 没有AI经验？先做个人项目，再写进简历
- AI-STAR比传统STAR多了"AI适配判断"和"边界认知"
- 简历必须附作品集链接，没有作品集等于没有差异化
- 30秒原则：招聘方决策很快，排版和关键词密度同样重要
