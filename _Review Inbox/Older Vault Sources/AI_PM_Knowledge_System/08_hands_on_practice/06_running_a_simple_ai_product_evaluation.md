📌 **一句话版本**
评测AI产品不是"感觉好不好"，而是用系统化的框架，
量化地比较准确性、速度、成本和用户体验。

> 🌐 **In English:**
> Evaluating AI products isn't about gut feeling — it's about systematically and quantitatively comparing accuracy, speed, cost, and user experience using a structured framework.

---

## 📖 这一节在讲什么

你可能遇到过这种情况：
老板问"我们的AI助手比竞品强吗？"
或者工程师问"换新模型效果更好了，能上线吗？"

如果你没有评测框架，只能凭感觉说"好像差不多"。
有了评测框架，你能说："在准确率上提升了12%，
延迟从3.2秒降到了1.8秒，成本下降了30%，建议上线。"

这一节教你搭建一个产品经理能用的AI评测体系，
不需要写代码，用表格和场景测试就能做。

> 🌐 **In English:**
> Without an evaluation framework, you can only say "it seems better." With one, you can say: "Accuracy improved 12%, latency dropped from 3.2s to 1.8s, cost reduced 30% — recommend launching." This section builds that capability.

---

## 🧠 为什么评测如此重要

AI产品不像传统软件，功能可以精确测试（按钮是否能点击）。
AI的输出是概率性的，每次都可能不同，
必须通过系统性评测来理解产品质量。

**评测能帮你回答这些问题：**
- 换新模型，整体效果有没有变好？
- 某个特殊场景下，AI会不会崩掉？
- 成本增加了10%，换来的质量提升值不值？
- 用户最不满意的是哪类回答？

> 🌐 **In English:**
> AI output is probabilistic — it varies every time. Systematic evaluation helps you understand product quality, compare models, find failure scenarios, and justify cost-quality tradeoffs.

## 🏗️ AI产品评测的基本框架

一个实用的AI产品评测框架包含四个维度：

### 维度1：准确性（Accuracy）

AI回答是否正确、是否符合事实？

| 指标 | 说明 | 测量方法 |
|------|------|---------|
| 答对率 | 有标准答案的题目 | 人工评分或自动对比 |
| 幻觉率 | 编造信息的比例 | 人工抽查验证 |
| 拒答率 | 该回答却不回答 | 统计比例 |

### 维度2：延迟（Latency）

用户等待AI响应的时间有多长？

```
关键指标：
- TTFT（Time to First Token）
  = 从发送请求到第一个字出现的时间
- 总响应时间 = 完整回答输出完成的时间

用户体验参考值：
< 1秒：极佳
1-3秒：可接受
3-5秒：开始影响体验
> 5秒：用户会大量流失
```

### 维度3：成本（Cost）

每次AI调用花了多少钱？

```
成本计算公式：
每次成本 = (输入token × 输入单价)
         + (输出token × 输出单价)

月度成本估算：
= 每次平均成本 × 每日调用量 × 30
```

### 维度4：用户体验（UX）

用户主观感受如何？
- 回答是否易于理解？
- 格式是否清晰？
- 有没有废话、重复、跑题？

> 🌐 **In English:**
> The four evaluation dimensions: **Accuracy** (is the answer correct?), **Latency** (how fast?), **Cost** (how much per call?), **UX** (is the answer helpful and readable?).

## 📋 设计一个简单的评测方案

**第一步：确定评测范围**

不要试图测所有场景，先聚焦3个最重要的用户场景。

示例（以AI客服产品为例）：
```
场景1：询问产品基本信息（高频）
场景2：投诉处理和情绪安抚（高风险）
场景3：退款退货流程询问（关键业务）
```

**第二步：准备测试集**

每个场景准备10-20个测试问题，
包含：
- 典型问题（大多数用户会问的）
- 边界问题（少见但重要的情况）
- 压力问题（刻意刁难AI的问题）

**第三步：制定评分标准**

在评测之前，先写清楚"什么算好答案"，
否则人工评分会不一致。

示例评分标准（1-5分）：

| 分数 | 含义 |
|------|------|
| 5分 | 完全正确，格式清晰，简洁有用 |
| 4分 | 基本正确，有小瑕疵 |
| 3分 | 大致方向对，但有明显不足 |
| 2分 | 部分正确，有明显错误 |
| 1分 | 完全错误或跑题 |

## 🎯 AI产品经理视角：评测的三个陷阱

**陷阱1：只测自己想测的场景**
评测集要包含"AI会失败的场景"，
不能只测AI表现好的地方。
建议：10%的测试问题刻意选择边界情况。

**陷阱2：用一个问题下结论**
AI的输出有随机性，
同一个问题要测3-5次，
取平均值，不能凭单次结果下结论。

**陷阱3：忽略业务场景的权重**
"产品信息查询"和"退款处理"
对用户来说重要性不一样。
评测时要给高频、高风险场景更高的权重，
而不是简单平均所有分数。

> 🌐 **In English:**
> Three evaluation traps: (1) Only testing scenarios where AI does well — include failure cases. (2) Judging from a single response — run each test 3-5 times. (3) Treating all scenarios equally — weight high-frequency and high-risk scenarios more heavily.

## ⚠️ 常见误解

**误解1：准确性最重要，其他不重要。**
不对。对于实时对话产品，
延迟超过5秒可能比准确性差更影响体验。
要根据产品场景决定哪个指标最重要。

**误解2：评测一次就够了。**
不对。AI模型会持续更新（有时悄悄更新），
而且你的业务场景也会变化。
建议建立定期评测机制，至少每季度一次。

**误解3：评测要很专业的工具才能做。**
不对。用Excel表格做评测，
专注在测试集设计和评分标准上，
已经能解决80%的实际问题。
