> 一句话版本：React 适合做可交互界面，Vite 适合快速开发和构建前端应用，它们常用于网站中的复杂应用部分。

## 这一节在讲什么

这一节解释 React 和 Vite 的基础。你会知道它们和 Astro 的关系，以及为什么一个个人网站里可能同时有内容站和可运行应用。

本节不要求你写 React，只要求你能看懂它在项目中的位置。

## 概念解释

`React` 是用于构建 UI 的 JavaScript library。它擅长处理状态、交互和可复用组件。

`Vite` 是现代前端开发和构建工具。它启动快，常用于 React、Vue 等前端应用。

Astro 更适合内容型页面，React/Vite 更适合交互丰富的应用。一个 workspace 可以同时包含两类东西。

## 现实例子

个人网站首页、项目页、文章页通常更像内容站。

一个专注训练工具、计时器、任务系统或交互测评，更像应用。它可能需要 React 处理状态和交互。

所以个人网站可以展示你是谁，也可以嵌入可运行 demo。

## Charlie 项目案例

Charlie 项目中 `apps/web` 是 Astro 主站，而 `apps/focus-training` 是可运行应用。`apps/web/astro.config.mjs` 里还有代理配置，把 `/apps/focus-training` 指向本地另一个端口。

这说明 Charlie 项目把「作品集叙事」和「可运行应用」分开。项目页解释思考，app 路径提供体验。

## 检查问题

1. React 是不是只能做整个网站？
2. Vite 主要解决内容写作问题，还是开发构建问题？
3. 为什么可运行应用适合和项目介绍分开？

## 常见误解

常见误解是认为学前端必须先学 React。对个人内容站来说，先理解页面、内容和部署同样重要。

另一个误解是把 Astro 和 React 当成只能二选一。真实项目可以按页面需求组合工具。

## 记住这些

- React 擅长交互 UI。
- Vite 擅长快速开发和构建。
- Astro 主站和 React 应用可以共存。
