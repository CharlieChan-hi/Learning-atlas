## 这一节在讲什么

这一节教你怎么使用 `Website Learning Studio`。它不是普通笔记文件夹，而是一套面向零基础学习者的课程型知识库。

你读它的目标不是记住每个术语，而是逐步建立三个能力：看懂网站项目目录、理解页面设计选择、知道网站如何从本地变成线上页面。

## 概念解释

知识库的作用是把复杂知识拆成稳定的路径。对于网站学习来说，最难的不是单个术语，而是术语之间的关系。

例如你可能听过 `HTML`、`CSS`、`JavaScript`、`Astro`、`React`、`Cloudflare`。如果这些词各自孤立，你会越看越乱。资料库的任务是把它们放回一条线里：内容如何写出来，页面如何组成，代码如何运行，网站如何上线。

学习时不要急着问「我什么时候能写完整网站」。更好的问题是：我今天能不能看懂一个目录？我能不能解释一个文件为什么存在？我能不能说出首页第一屏为什么这样排？

## 现实例子

把网站学习想成学会看一栋房子的图纸。你不需要第一天就会盖房子，但你需要知道客厅、卧室、水管、电线分别在哪里。

网站项目也是一样。`content` 像文字和图片素材，`components` 像可重复使用的积木，`pages` 像每个房间的入口，`package.json` 像项目说明书和命令表。

当你能把这些部分对应起来，你就不会被一堆文件吓住。

## Charlie 项目案例

在只读观察中，`/Users/mac/Desktop/charliechanstudio.com` 是一个 pnpm workspace 项目。根目录的 `pnpm-workspace.yaml` 指向 `apps/*` 和 `packages/*`，意思是这个项目把应用和共享工具分开管理。

`apps/web` 是 Astro 主站，`packages/content` 负责读取项目内容，`packages/lib` 存放路线和 SEO 相关工具，`content/projects` 放项目内容文件。

你在这个资料库里看到 Charlie 项目案例时，要把它当作真实地图，而不是要立刻修改的对象。本次学习库不会改 Charlie 项目。

## 检查问题

1. 这个资料库的目标是教你改正式网站，还是教你理解网站？
2. 为什么要先理解目录，而不是一上来写页面？
3. 读到不懂的术语时，应该立刻卡住，还是先记录后继续？

## 常见误解

常见误解是认为学习网站必须从写代码开始。对零基础来说，先建立结构感更重要。你要先知道代码放在哪里、内容怎么流动、页面为什么这样组织。

另一个误解是认为资料越长越好。真正有用的资料不是堆字数，而是让你能复述、能练习、能回到真实项目里验证。

## 记住这些

- 先建立地图，再学习细节。
- 每次只解决一个层级的问题。
- Charlie 项目只作为学习案例，不作为本次修改对象。
