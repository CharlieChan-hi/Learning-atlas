📌 **一句话版本**
> AI是你的"随身语言顾问"——帮你把意思说得更准确、
> 更专业、更适合对方——但你还是那个最懂上下文的人。

> 🌐 **In English:**
> AI is your on-demand language coach — helping you say things
> more precisely, professionally, and audience-appropriately.
> But you're still the one who knows the full context.

---

## 📖 这一节在讲什么

本节覆盖PM日常高频沟通场景：
润色邮件/消息、准备会议议程和纪要、
辅助方案对比和决策、知识管理整理，
以及跨语言沟通支持。
每个场景提供可直接复制使用的Prompt模板。

---

## ⚠️ 使用AI辅助沟通的原则

在用AI润色沟通内容前，请明确以下几点：

1. 保持你的声音：AI改完后，读一遍确认是"你"说话
2. 核查事实：AI不了解你们公司内部情况
3. 注意语气对齐：对上级、平级、下属语气不同
4. 不要过度润色：太正式的内部邮件反而显得疏离

> 🌐 **In English:**
> Keep your authentic voice. AI polish should sound like you,
> not like a corporate press release. Adjust tone for audience:
> manager, peer, direct report, or external partner.

## 🧠 场景一：润色邮件和消息

### 适用情境

你写好了邮件草稿，但感觉表达不够清晰、
语气不太对、或者需要翻译成英文。

### Prompt模板 A：润色中文邮件

```
请帮我润色以下邮件草稿，要求：
- 保持原意不变
- 语气：[专业/友好/直接/谦逊]
- 对象：[上级/平级/外部合作方/研发团队]
- 重点突出：[你最想对方记住的一句话]

邮件草稿：
[粘贴你的草稿]
```

### Prompt模板 B：翻译+润色成英文邮件

```
请将以下中文邮件翻译成英文，要求：
- 语气：professional but friendly
- 对象：[外部合作方/英语母语同事/海外用户]
- 如果有口语表达，请调整为邮件用语

原文：
[粘贴中文内容]

另外请指出：原文中是否有任何
在英文邮件中不常见或需要解释的表达？
```

### Prompt模板 C：快速改写Slack/微信消息

```
帮我改写以下消息，使其更清晰简洁：
- 当前长度：[太长/太短/刚好]
- 目标：[让对方知道我需要什么/传递信息/请求帮助]
- 语气：[非正式但专业]

原消息：
[粘贴消息]
```

### 操作步骤

```
第1步：先自己写出草稿，不要一开始就让AI写
       （你的草稿保留了真实意图）
第2步：选择对应的Prompt模板
第3步：填入语气/对象/重点等参数
第4步：AI输出后通读一遍，
       确认每句话都是你想说的
第5步：修改1-2处AI改得不自然的地方
```

💡 **提示：永远先写草稿，再让AI润色**

从空白让AI直接写邮件，结果往往太模板化。
先写出哪怕粗糙的草稿，AI润色效果好10倍。

## 🧠 场景二：准备会议议程和纪要

### 适用情境

你需要组织一个需求评审会、方案讨论会，
或者会后需要快速整理会议纪要发给团队。

### Prompt模板 A：生成会议议程

```
请帮我生成一份会议议程，信息如下：

会议类型：[需求评审/方案讨论/周例会/复盘]
参会人：[产品/研发/设计/运营/管理层]
会议时长：[60分钟/90分钟]
核心目标：这次会议结束时，
          我希望达成什么决定或共识？
          [填写1-2条]
背景：[简要说明当前状况]

请输出：
1. 议题列表（含建议时长分配）
2. 每个议题的"期望产出"（结论/决策/行动项）
3. 会前需要参会者准备的材料清单
```

### Prompt模板 B：整理会议纪要

```
以下是我的会议笔记（可能比较凌乱），
请帮我整理成规范的会议纪要：

输出格式：
1. 会议基本信息（时间/参会人/主题）
2. 讨论要点（每个议题用小标题）
3. 决议事项（明确的结论）
4. 行动项清单（负责人+截止时间）
5. 遗留问题（未解决的需跟进）

会议笔记：
[粘贴你的原始笔记]
```

### 操作步骤

```
会前准备：
  第1步：用Prompt A生成议程框架
  第2步：修改成符合实际情况的版本
  第3步：会前24小时发给参会者

会后整理：
  第4步：会中记录关键讨论点和结论
  第5步：会后用Prompt B粘贴笔记整理
  第6步：发送前核对行动项分工是否准确
```

### 🎯 AI产品经理视角

好的会议议程是"以结论为导向"的：
每个议题都对应一个需要产出的决定。
用AI生成议程时，关键是你想清楚了：
这个会的核心决策是什么？

> 🌐 **In English:**
> A good agenda is outcome-oriented: each item maps to a decision.
> The key when using AI to draft agendas is knowing in advance:
> what is the one core decision this meeting must produce?

## 🧠 场景三：方案对比与决策辅助

### 适用情境

你面临多个方案（技术选型、功能优先级、
用户分层策略等），需要快速梳理利弊，
帮助团队或上级做决策。

### Prompt模板 A：结构化方案对比

```
我有以下[N]个方案需要对比，
请帮我生成结构化对比分析：

方案概述：
- 方案A：[简要说明]
- 方案B：[简要说明]
- 方案C：[简要说明]

我的评估维度（按重要性排序）：
1. [维度1，如：开发成本]
2. [维度2，如：用户体验影响]
3. [维度3，如：可维护性]
4. [维度4，如：上线时间]

请输出：
1. 对比表格（行=维度，列=方案）
2. 每个方案的最大优势和最大风险
3. 你认为最均衡的方案及理由
4. 我可能没有考虑到的因素
```

### Prompt模板 B：辅助写决策建议

```
我需要向[对象：老板/团队/评审委员会]
汇报以下决策建议，请帮我润色：

结论：我推荐选择方案[X]
理由：
1. [理由1]
2. [理由2]
3. [理由3]
风险：[主要风险]
缓解措施：[如何降低风险]

要求：
- 语气：自信但不武断
- 长度：适合口头汇报，3-5分钟
- 格式：可以用要点，便于我记忆
```

### 💡 提示：让AI扮演"魔鬼代言人"

```
专用Prompt：
请以反对者的角度，
指出我的方案[X]最大的3个漏洞，
并给出针对每个漏洞的应对策略。
```

这个技巧能帮你在汇报前提前发现弱点。

## 🧠 场景四：知识管理与笔记整理

### 适用情境

你有一堆零散的调研笔记、文章收藏、
会议记录，需要整理成有结构的知识库，
方便以后快速检索和引用。

### Prompt模板 A：整理零散笔记

```
以下是我关于[主题]的零散笔记，
请帮我：
1. 提取关键信息点（去除重复）
2. 按逻辑分组（建议3-5个组）
3. 每组用一句话总结核心观点
4. 标出你认为最重要的3条洞察

笔记原文：
[粘贴笔记内容]
```

### Prompt模板 B：从长文章提炼要点

```
以下是一篇文章，请帮我提炼：
1. 核心观点（1句话）
2. 关键论据（3-5条）
3. 适用场景（这个观点在什么情况下有用）
4. 我作为产品经理，
   可以从这篇文章学到什么？

文章内容：
[粘贴文章]
```

### Prompt模板 C：建立个人知识卡片

```
请将以下内容整理成一张知识卡片：

主题：[填写]
一句话定义：[AI帮你提炼]
核心要点：[3-5条]
类比解释：[用更简单的比喻]
PM应用场景：[在工作中怎么用]
相关概念：[链接到其他知识]

原始材料：
[粘贴内容]
```

### 操作步骤

```
建立知识库的推荐流程：

第1步：用Notion/Obsidian建立主题文件夹
第2步：收到新内容时，用Prompt B提炼要点
第3步：整理成知识卡片（Prompt C）存入库
第4步：每月用AI做一次"知识连接"：
  问AI：这些内容之间有什么关联？

定期维护Prompt：
请帮我回顾以下知识条目，
找出互相矛盾或可以合并的部分：
[粘贴多条笔记]
```

## 🧠 场景五：跨语言沟通辅助

### 适用情境

你需要和英文母语的同事/合作方沟通，
或者需要处理英文产品文档、需要用英文
做产品演示、回复英文反馈。

### Prompt模板 A：检查英文表达是否地道

```
请检查以下英文表达，
告诉我：
1. 是否有语法错误
2. 是否有不自然或过于书面的表达
3. 给出更地道的替换建议
4. 标注哪些是中式英语（Chinglish）

原文：
[粘贴你的英文]
```

### Prompt模板 B：英文邮件快速起草

```
请帮我起草一封英文邮件：

收件人背景：[英语母语/非母语/技术背景/商务背景]
邮件目的：[告知/请求/跟进/感谢/提问]
核心信息：[1-2句话说明你要传达什么]
语气：[formal/semi-formal/casual]

要求：
- 200字以内
- 主题行也一起写
- 结尾方式与语气匹配
```

### Prompt模板 C：理解英文文档中的模糊表达

```
以下英文段落中有一些我不确定意思的表达，
请帮我：
1. 解释整段的意思
2. 解释我标出的具体词句：[标出词句]
3. 告诉我这个表达在商务场景中的含义
4. 如果我要用中文向团队传达，
   应该怎么说？

英文原文：
[粘贴内容]
```

### 🎯 AI产品经理视角

跨语言沟通中，AI最有价值的地方不是翻译，
而是帮你理解"语气"和"潜台词"：
对方说"That's interesting"到底是真感兴趣还是礼貌拒绝？
这类文化语境问题，AI能给你靠谱的参考。

> 🌐 **In English:**
> AI's biggest value in cross-language communication isn't translation
> — it's interpreting tone and subtext.
> "That's interesting" or "We'll consider it" can mean very different
> things. AI can help you read between the lines.

**练习1：邮件润色对比**
找一封你之前发过的邮件，
用Prompt A润色，对比前后版本，
看看AI改了什么，你同意多少。

**练习2：会议议程生成**
下一次你要组织会议时，
先用AI生成一份议程，
再和你平时准备的议程对比，
看看有没有漏掉的要素。

**练习3：知识卡片制作**
选一篇你最近读的文章，
用Prompt B提炼要点，
再用Prompt C制作成知识卡片，
存入你的个人知识库。

## 📝 记住这些

1. 沟通场景中，AI帮你打磨表达，
   但你的判断和判断力不能外包
2. 会议议程的核心价值是明确"产出是什么"，
   不只是列时间表
3. 方案对比时，让AI扮演反对者，
   是发现盲点的高效方法
4. 英文沟通中，语气和文化语境比语法更重要
5. 知识管理的关键是"整理后要能找到并用到"

> 🌐 **In English — Key Takeaways:**
> AI is most powerful in daily communication when used as a refinement
> layer, not a replacement for thinking. Draft first, polish second.
> For meetings: focus AI on clarifying decisions, not just formatting.
> For cross-language work: use AI to check tone and cultural fit,
> not just grammar. Your judgment about context remains the core skill.
