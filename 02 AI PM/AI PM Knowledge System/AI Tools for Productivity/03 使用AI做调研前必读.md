> 但真正的洞察仍需要你用产品直觉来完成最后一公里。

> 🌐 **In English:**
> AI acts as your junior analyst — quickly building industry frameworks,
> but the real insight still requires your product intuition to close
> the last mile.

---

## 📖 这一节在讲什么

本节聚焦4个PM高频调研场景：
快速了解一个陌生行业、分析竞品产品逻辑、
处理简单数据分析（CSV上传+提问）、
生成数据可视化的语言描述。
每个场景提供Prompt模板和实际操作步骤。

---

## ⚠️ 使用AI做调研前必读

AI竞品调研存在以下局限，务必知晓：

1. 知识截止日期：AI不了解产品最新版本功能
2. 细节失真：AI会混合多个版本的功能描述
3. 数据虚构：市场份额、DAU等数字需独立核实
4. 主观色彩：AI对"哪个更好"的判断缺乏使用体验

> 🌐 **In English:**
> AI has a knowledge cutoff, may mix product versions, fabricate numbers,
> and lacks real usage experience. Always verify key claims independently.

## 🧠 场景一：快速了解一个陌生行业

### 适用情境

你被分配到一个新赛道（如：AI教育、工业SaaS、出海电商），
需要在1-2天内建立基本行业认知，准备好和老板/同事对话。

### Prompt模板

```
你是一位资深行业分析师。
请用以下结构帮我快速了解[行业名称]：

1. 行业定义（2-3句话）
2. 市场规模和增长趋势（概述，不必精确数字）
3. 产业链上下游（用简单层次结构描述）
4. 主要玩家分类（按商业模式或用户群分组）
5. 行业核心矛盾/痛点（3条以内）
6. 近1-2年最值得关注的变化

输出格式：每部分小标题+3-5行正文
```

### 输入示例

```
请用以上结构帮我快速了解：
工业设备预测性维护（Predictive Maintenance）行业
```

### 输出示例（节选）

```
1. 行业定义
通过传感器、数据采集和机器学习模型，
预测工业设备故障时间，
从"故障后修"转向"故障前维"。

2. 市场规模
全球2023年约40-60亿美元，
预计2028年达100亿以上（CAGR约15-20%）。
主要驱动力：制造业数字化、碳排放压力。
...
```

### 操作步骤

```
第1步：用上述Prompt获取行业框架
第2步：追问你最感兴趣的1-2个子话题
       （如：国内主要竞争格局是什么？）
第3步：用AI输出内容作为关键词，
       去36氪/36kr/艾瑞/行研报告做二次核实
第4步：整理成自己的行业一页纸
```

💡 **提示：行业了解要分层**

不要一次要求AI输出所有信息，分层提问效果更好：
先要全局框架 → 再深挖某个维度 → 最后聚焦竞争格局。

## 🧠 场景二：分析竞品产品逻辑

### 适用情境

你需要对比3-5个竞品的核心功能、用户流程、
商业模式，用于立项报告或产品规划参考。

### Prompt模板 A：功能对比框架

```
请帮我分析以下产品的核心功能逻辑，
用结构化方式输出：

产品：[产品A] vs [产品B] vs [产品C]
分析维度：
1. 核心用户群（谁在用）
2. 主要使用场景（解决什么问题）
3. 关键功能模块（列出3-5个核心模块）
4. 差异化亮点（各自最突出的地方）
5. 明显不足（已知的用户投诉/限制）

输出格式：对比表格，每列一个产品
```

### Prompt模板 B：深挖单个竞品逻辑

```
请帮我分析[产品名称]的产品逻辑：

1. 它的核心用户旅程是什么？
   （从用户第一次打开到完成核心任务）
2. 它的增长飞轮是什么？
   （用户如何带来更多用户）
3. 它的商业化路径是什么？
   （免费+付费的边界在哪里）
4. 它的护城河在哪里？
   （为什么用户难以迁移走）

请诚实说明哪些内容你不确定或可能过时
```

### 操作步骤

```
第1步：先用Prompt A生成总览对比表
第2步：标记出你感兴趣的差异点
第3步：用Prompt B深挖1-2个重点竞品
第4步：结合App Store评论/产品截图做补充
第5步：用AI帮你整理成竞品分析文档初稿
```

### 🎯 AI产品经理视角

竞品分析AI能帮你完成的是**结构化整理**，
不能替代你自己**亲手用产品**的体验。
最有价值的竞品洞察往往来自："我用了之后发现..."，
AI给你的是框架，你给框架填的是真实体验。

> 🌐 **In English:**
> AI helps you structure competitive analysis, but can't replace
> hands-on product experience. The best insights come from
> "I used it and noticed..." — AI gives you the framework,
> you fill it with real observations.

## 🧠 场景三：用AI处理简单数据分析

### 适用情境

你有一份CSV数据（用户行为日志、问卷结果、
功能使用统计），想快速理解数据在说什么，
但不想写SQL或Python。

### 支持上传CSV的工具

| 工具 | 上传CSV | 自动分析 | 生成图表 |
|------|---------|---------|---------|
| ChatGPT（Plus） | 支持 | 支持 | 支持 |
| Claude | 支持 | 支持 | 文字描述 |
| Gemini Advanced | 支持 | 支持 | 支持 |

### Prompt模板：上传CSV后提问

```
我上传了一份数据文件，请帮我：

1. 描述数据的基本结构
   （有多少行、多少列、每列含义）
2. 找出明显的数据质量问题
   （空值、异常值、重复行）
3. 回答以下问题：
   [你的具体分析问题，如：
    "哪个功能的使用频次最高？"
    "用户留存在哪一天下降最多？"]
4. 给出3条最重要的数据观察结论
```

### 实际操作步骤

```
第1步：清理数据
  - 删除敏感信息（用户ID脱敏）
  - 确认列名清晰可读
  - 文件大小控制在10MB以内

第2步：上传到ChatGPT（Data Analysis模式）
  或者将数据粘贴到Claude对话框

第3步：先发送"描述这份数据的结构"
  确认AI正确理解了数据

第4步：逐个提问，避免一次提太多问题

第5步：要求AI输出分析结论，而不只是数字
```

### 💡 提示：让AI解释数字背后的含义

不要只问"第3列平均值是多少"，
要问"第3列的平均值意味着什么，对我们产品有什么影响"。
AI在**解释意义**上比纯计算更有价值。

## 🧠 场景四：用AI生成数据可视化描述

### 适用情境

你有数据结论，需要在PPT或报告中用语言
描述图表趋势，或者需要指导设计师/研发
制作图表时，用语言规格描述清楚。

### Prompt模板：描述图表趋势

```
以下是我的数据：
[粘贴关键数据，如：
  1月：1200用户，2月：1450用户，
  3月：1380用户，4月：1620用户]

请帮我：
1. 用1句话描述整体趋势
2. 指出最值得关注的变化点和可能原因
3. 写一段适合放在报告中的数据解读
   （3-4句话，非技术人员可读懂）
```

### Prompt模板：指导制作图表

```
我需要制作一个图表，请帮我写规格说明：

数据类型：[折线图/柱状图/饼图/散点图]
数据内容：[简要描述]
核心信息：我想让读者看完图后记住什么？
受众：[内部评审/对外汇报/技术团队]

请输出：
1. 推荐图表类型及理由
2. 横轴/纵轴应该放什么
3. 需要标注的关键数据点
4. 图表标题建议（10字以内）
```

### 🎯 AI产品经理视角

数据可视化的核心是**"一张图说清一件事"**。
让AI帮你提炼"这份数据最重要的一个结论是什么"，
比让AI列出所有数据发现更有用。

> 🌐 **In English:**
> The core of data visualization is "one chart, one message."
> Ask AI to distill "what's the single most important finding"
> rather than listing every observation.

## 📝 记住这些

1. AI做竞品调研的价值：**快速结构化**，不是"告诉你真相"
2. 数据分析时先问"这数据说的对吗"，再问"说明了什么"
3. 所有AI生成的数字（市场规模、增长率）必须独立核实
4. 行业调研：AI给框架，你补充第一手信息
5. 竞品分析的最终价值来自你自己用产品后的判断
