## 你要立刻记住的 12 条规则

1. 每次开始前先看 `git status --short --branch`。
2. 提交前一定看 `git diff`。
3. 初学阶段优先 `git add 文件名`，少用 `git add .`。
4. commit 信息写“为什么改”。
5. commit 是本机保存，push 是上传云端。
6. B 电脑要拿到 A 电脑成果，A 必须先 push，B 再 pull。
7. GitHub private repository 适合你的个人网站早期阶段。
8. Cloudflare Pages 未来最好连接 GitHub 仓库部署。
9. 已经 push 的历史，优先用 revert，不要随便 reset。
10. 不懂时不要运行 `reset --hard`、`clean -fd`、`push --force`。
11. `.env`、密钥、账号密码永远不要提交。
12. 真实项目动手前，先用练习仓库练命令。

## 提交前检查清单

每次准备 commit 前，按这个顺序做：

```text
□ 我知道这次修改的目的是什么
□ 我看过 git status
□ 我看过 git diff
□ 我只 add 了相关文件
□ 没有 add .env 或密钥
□ 没有 add 大型临时文件
□ 如果是代码改动，已经运行必要测试
□ commit message 写清楚为什么改
```

## 常用命令速查

| 目的 | 命令 |
| --- | --- |
| 看状态 | `git status --short --branch` |
| 看未暂存改动 | `git diff` |
| 看已暂存改动 | `git diff --staged` |
| 暂存某文件 | `git add 文件名` |
| 创建提交 | `git commit -m "说明原因"` |
| 看历史树 | `git log --oneline --graph --decorate --all` |
| 取消暂存 | `git restore --staged 文件名` |
| 撤回未提交修改 | `git restore 文件名` |
| 上传远程 | `git push` |
| 拉取远程 | `git pull` |
| 克隆仓库 | `git clone 仓库地址` |

## 危险命令提醒

初学阶段不要随便运行：

```bash
git reset --hard
git clean -fd
git push --force
```

它们可能导致：

- 未提交内容丢失；
- 新文件被删除；
- GitHub 远程历史被覆盖。

## 最后提醒

Git 对你来说不是为了“像程序员一样炫技”，而是为了让你的个人网站有稳定的长期维护方式：

```text
能保存历史
能看差异
能回滚
能跨电脑
能上云端
能连接部署
```

你现在最先掌握四件事就够：

```text
git status：我现在在哪、哪些文件变了
git diff：具体变了什么
commit：保存一个本地版本点
push/pull：和 GitHub 云端同步
```
