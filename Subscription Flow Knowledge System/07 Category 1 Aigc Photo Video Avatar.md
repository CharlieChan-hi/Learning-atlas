> One-line version: Different app categories need different subscription journeys because users buy different kinds of value.

---

## Category 1: AIGC Photo / Video / Avatar

### 用户为什么买

用户买的是：

- 结果完整性。
- 高清导出。
- 无水印。
- 更多模板。
- 更快生成。
- 更多生成次数。

### 推荐流程

```text
Premium loading
  -> result-first onboarding
  -> template preview
  -> pre-paywall result
  -> paywall
  -> payment
  -> unlocked result
```

### 为什么这么设计

AI 创作类的核心价值很视觉。用户不需要先理解功能，他需要先看到“我的照片会变成什么”。

### 不建议

- 重问卷。
- 先讲 AI 技术。
- 先登录。
- 先索取通知权限。

## Category 2: AI Assistant / Productivity

### 用户为什么买

用户买的是：

- 更强模型。
- 更大额度。
- 更快速度。
- 更长上下文。
- 文件处理。
- 专业工作流。

### 推荐流程

```text
light loading
  -> direct task input
  -> first useful output
  -> advanced feature or quota paywall
```

### 为什么这么设计

效率工具用户通常带着任务来。越快让他完成任务，越容易建立付费理由。

## Category 3: Fitness / Health / Nutrition

### 用户为什么买

用户买的是：

- 个性化计划。
- 长期陪伴。
- 进度追踪。
- 自律感。
- 专业可信度。

### 推荐流程

```text
welcome
  -> goal questionnaire
  -> body / habit / time constraints
  -> personalized plan ready
  -> subscription paywall
```

### 为什么这么设计

用户不是买一次功能，而是买“我能不能变好”。问卷可以建立“这个计划是为我做的”的感觉。

## Category 4: Learning / Language / Education

### 用户为什么买

用户买的是：

- 学习路径。
- 课程库。
- 进度。
- 每日练习。
- 更高阶内容。

### 推荐流程

```text
goal selection
  -> level check
  -> first easy lesson
  -> path preview
  -> paywall
```

### 为什么这么设计

学习类的价值不是看见功能，而是感到“我已经开始学了”。

## Category 5: Meditation / Sleep / Mental Wellness

### 用户为什么买

用户买的是：

- 安定感。
- 陪伴感。
- 睡眠改善。
- 情绪管理。
- 课程库。

### 推荐流程

```text
calm welcome
  -> emotional goal
  -> first recommendation
  -> short experience
  -> subscription
```

### 为什么这么设计

这类用户可能带着压力进入。页面越像强销售，越容易破坏信任。

## Category 6: Finance / Security / Insurance

### 用户为什么买

用户买的是：

- 风险降低。
- 专业信任。
- 数据安全。
- 更清晰的管理能力。
- 关键时刻的保障。

### 推荐流程

```text
trusted loading
  -> security explanation
  -> value demo
  -> plan explanation
  -> payment
```

### 为什么这么设计

这类产品信任比冲击力更重要。少炫技，多透明。

## Category 7: Content / Music / Video / Reading

### 用户为什么买

用户买的是：

- 无广告。
- 无限访问。
- 离线。
- 独家内容。
- 更顺的长期体验。

### 推荐流程

```text
content preview
  -> interest selection
  -> first content experience
  -> premium paywall
```

### 为什么这么设计

内容产品卖的是长期消费体验，不是一次功能。

## Category 8: Social / Dating / Community

### 用户为什么买

用户买的是：

- 曝光。
- 更好筛选。
- 更多互动机会。
- 社交结果。
- 安全感。

### 推荐流程

```text
profile setup
  -> preference
  -> first matches / content
  -> premium feature paywall
```

### 为什么这么设计

用户必须先相信“这里有真实的人、真实机会、真实安全感”。

## Quick Decision Table

| Category | Onboarding | Paywall Trigger |
|---|---|---|
| AIGC | Result-first | After preview / before HD export |
| Fitness | Quiz-first | Personalized plan ready |
| Learning | Goal + first lesson | Path preview / premium lesson |
| Productivity | Direct task | Quota / advanced feature |
| Finance | Trust-first | Plan or premium control |
| Content | Content preview | Premium experience difference |
