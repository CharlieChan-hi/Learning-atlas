## 🧠 核心概念：视觉动线到底是什么

视觉动线指用户看页面时，眼睛会沿着内容、标题、图片、色块、按钮等元素自然移动的路线。设计师真正要控制的，不是用户每一秒看哪里，而是让重要内容更容易先被看见，让次要内容不会抢戏，让行动按钮出现在用户已经理解信息之后。

换句话说，视觉动线是页面里的“注意力路线图”。如果路线清楚，用户会觉得页面顺；如果路线混乱，用户会觉得信息很多但抓不到重点。

> 🌐 **English focus:**
> Remember these terms: **visual flow**, **visual hierarchy**, **attention path**, **key information**, **primary action**.

| 术语 | 中文理解 | 设计语境里的用法 |
|---|---|---|
| visual flow | 视觉动线 | The page has a clear visual flow. |
| visual hierarchy | 视觉层级 | Use size and contrast to create visual hierarchy. |
| attention path | 注意力路径 | Guide the user's attention path from title to CTA. |
| key information | 关键信息 | Put key information where users look first. |
| primary action | 主要行动 | Make the primary action easy to find. |

## 🧭 模型一：古腾堡原则

古腾堡原则适合解释从左到右、从上到下的阅读习惯。它把页面分成四个视觉区域：左上通常最先被看见，右下通常是视觉终点。

这不是说所有页面都必须照这个模板摆，而是提醒我们：如果页面是卡片、列表项、服务信息、交易决策这类结构，左上和右下天然承担更高责任。

![古腾堡原则标题图](assets/gutenberg-title.png)

![古腾堡四象限示意图](assets/gutenberg-quadrants.png)

| 区域 | 中文判断 | 推荐放什么 |
|---|---|---|
| Primary Optical Area | 第一视觉区，通常是左上 | 标题、核心信息、用户最需要先知道的内容 |
| Final Optical Area | 最终视觉区，通常是右下 | CTA、确认按钮、下一步行动 |
| Strong Follow Area | 强休息区，通常是右上 | 辅助信息、标签、补充状态 |
| Weak Follow Area | 弱休息区，通常是左下 | 低优先级说明、非关键内容 |

> 🌐 **English focus:**
> Key terms: **primary optical area**, **final optical area**, **CTA**, **decision point**, **secondary information**.

### 🎯 如何用在移动端卡片

文章里的 58 到家卡片案例很典型：标题和主要信息放在用户先看的区域，行动按钮放在视觉收束的位置。这样用户不用重新找按钮，理解完信息就能顺手行动。

![古腾堡原则移动端卡片案例 1](assets/gutenberg-mobile-card-1.png)

![古腾堡原则移动端卡片案例 2](assets/gutenberg-mobile-card-2.png)

你可以用一句英文记这个设计动作：

> Place the key information first, then lead users to the primary action.

## 🧠 模型二：F 型视觉模型

F 型视觉模型来自眼动追踪研究，常出现在文字密集页面里。用户会先横向扫顶部，再下移扫较短的一行，最后沿左侧纵向扫描。

这背后的逻辑很简单：用户会偷懒。不是贬义，而是人会自然选择最省力的阅读方式。所以设计师要用标题、加粗、列表、分组、间距去帮助用户扫读，而不是期待用户逐字读完。

![F 型视觉模型示意图](assets/f-shaped-pattern.png)

| F 型特征 | 设计启发 |
|---|---|
| 首行更容易被看见 | 标题和首句要承担总结功能 |
| 左侧词语更容易被看见 | 关键词尽量前置 |
| 用户会快速纵向扫描 | 用小标题、分段和列表制造停顿点 |
| 页面缺少层级时更容易 F 型扫读 | 用视觉层级打断无效扫读 |

> 🌐 **English focus:**
> Key terms: **F-shaped pattern**, **eye-tracking**, **scan**, **skim**, **content-heavy page**, **left-side scanning**.

### 🎯 文章里的移动端案例

58 同城商家版线索订阅页使用单排卡片，而不是双排卡片，是为了让标题和线索信息更容易被连续扫描。它的重点不是“排得满”，而是让用户能稳定读到线索标题、状态和重要信息。

![F 型移动端案例：线索订阅页面](assets/f-pattern-mobile-case.png)

适合记住的英文表达：

> Users scan before they read.

这句话很重要。它提醒你：用户通常先扫，再决定要不要认真读。

## 🧩 模型三：Z 型视觉模型

Z 型视觉模型更适合简洁页面，比如落地页、活动页、介绍页、首页首屏。用户先看顶部，从左到右建立认知；再斜向下移动；最后在底部完成判断或行动。

Z 型页面最怕元素零散。顶部、中段、底部必须有明显承接关系，否则用户会觉得页面只是把东西堆在一起。

![Z 型视觉模型示意图](assets/z-pattern.png)

> 🌐 **English focus:**
> Key terms: **Z-pattern**, **landing page**, **hero section**, **supporting content**, **conversion**, **call to action**.

你可以这样理解：

| Z 型路径 | 中文任务 | 常见设计元素 |
|---|---|---|
| 顶部横向 | 建立第一认知 | 品牌、标题、导航、核心卖点 |
| 斜向移动 | 承接信息 | 图片、产品说明、价值证明 |
| 底部横向 | 推动行动 | CTA、注册、购买、预约、查看详情 |

常用英文句：

> Use the Z-pattern to guide users from awareness to action.

## 🔥 X 上高互动观点如何融入这篇文章

这里不把 X 帖子当作附录堆在后面，而是把它们变成对文章观点的补强。

### 1. UI 不是玄学，而是规则

Adham Dannaway 的高互动帖提到，UI design 看起来像一种魔法，但很多效果来自 logical rules or guidelines。这个观点非常适合补充视觉动线：视觉动线不是凭感觉摆元素，而是把信息优先级变成可见的秩序。

> 🌐 **English focus:**
> Useful phrase: **repeatable design rules**  
> Meaning: 可复用的设计规则。

可以记住这句话：

> Good UI is built on repeatable design rules.

### 2. 视觉层级就是“重要性排序”

Adham 另一条关于 clear visual hierarchy 的帖子强调：按信息重要性来呈现内容，让更重要的元素更突出。这句话几乎就是视觉动线的执行标准。

落到设计里，就是三个问题：

1. 用户应该先看什么？
2. 用户看完之后应该看哪里？
3. 用户最后应该做什么？

> 🌐 **English focus:**
> Useful terms: **clear visual hierarchy**, **order of importance**, **prominent elements**.

### 3. 基础设计原则会影响用户怎么扫读

Sachin Ramje 的高赞设计原则帖适合补充 F 型模型。对比、层级、对齐、重复这些原则，不只是“让页面好看”，它们会改变用户扫读页面的方式。

| 英文原则 | 中文理解 | 对视觉动线的影响 |
|---|---|---|
| contrast | 对比 | 让重点跳出来 |
| hierarchy | 层级 | 告诉用户先看什么后看什么 |
| alignment | 对齐 | 降低扫描成本 |
| repetition | 重复 | 让页面模式更可预测 |
| spacing | 间距 | 给信息分组，制造呼吸感 |

一句好用的英文：

> Design principles shape how users scan a page.

## 📝 记住这些

视觉动线的本质是组织注意力。古腾堡原则、F 型模型、Z 型模型都不是固定模板，而是帮助你判断页面应该怎么安排信息。

如果你想学英文，最该记的不是整段翻译，而是这些设计语境里的词：**visual flow**, **visual hierarchy**, **scanning pattern**, **CTA**, **contrast**, **alignment**, **spacing**, **conversion**。
